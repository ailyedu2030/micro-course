# 数据同步接口规范 v1.0

> **状态**: P2 - 需对接教务系统，当前不可用
>
> **最后更新**: 2026-06-12

---

## 1. 概述

数据同步功能用于将外部教务系统中的用户、课程、班级等基础数据同步到微课平台。当前状态为**占位规范**，待与教务系统对接后方可实施。

---

## 2. 接口规范

### 2.1 用户数据同步

#### 请求
```
POST /api/sync/users
Content-Type: application/json
```

```json
{
  "timestamp": "2026-06-12T10:00:00Z",
  "operation": "FULL" | "INCREMENTAL",
  "data": [
    {
      "studentNo": "2021001234",
      "username": "zhangsan",
      "realName": "张三",
      "gender": "MALE" | "FEMALE" | "SECRET",
      "email": "zhangsan@example.com",
      "phone": "13800138000",
      "departmentCode": "CS",
      "majorCode": "CS2021",
      "classCode": "CS202101",
      "role": "STUDENT" | "TEACHER"
    }
  ]
}
```

#### 响应
```json
{
  "code": 200,
  "message": "ok",
  "data": {
    "successCount": 100,
    "failCount": 0,
    "failures": []
  }
}
```

## 3. 错误码

| 错误码 | 说明 |
|--------|------|
| 13001 | 教务系统不可用 |
| 13002 | 数据格式错误 |
| 13003 | 同步超时 |
| 13004 | 部分数据同步失败 |

---

## 4. 同步策略

1. **全量同步**: 每月1日凌晨执行，清洗后重新导入
2. **增量同步**: 每日凌晨执行，仅同步新增/变更数据
3. **实时同步**: 通过消息队列实现变更推送（待规划）

---

## 5. 注意事项

- 所有同步接口需要管理员权限
- 同步前需校验数据格式和业务规则
- 失败数据需记录并支持重试
- 同步过程需记录操作日志
