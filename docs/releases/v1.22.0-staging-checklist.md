## 2. 部署 v1.22.0 到 staging

### 2.1 后端 jar 部署

```bash
# 1. 上传新 jar（从 CI artifacts 或本地构建）
scp micro-course-api/target/micro-course-api-1.0.0.jar.v1.22.0 \
    <staging_user>@<staging_host>:/tmp/

# 2. 替换运行中的 jar
ssh <staging_user>@<staging_host> \
    "docker cp /tmp/micro-course-api-1.0.0.jar.v1.22.0 \
    micro-course-micro-course-api-1:/app/app.jar"

# 3. 优雅重启（不是 kill -9）
ssh <staging_user>@<staging_host> \
    "docker exec micro-course-micro-course-api-1 kill -s HUP 1"

# 4. 等待启动
sleep 15

# 5. 健康检查
curl -s http://staging-host:8080/actuator/health
# 预期: {"status":"UP"}
```

### 2.2 前端 dist 部署

```bash
# 1. 上传新 dist（从 CI artifacts 或本地构建）
scp -r micro-course-admin/dist/. \
    <staging_user>@<staging_host>:/opt/micro-course/micro-course-admin/dist/.v1.22.0/

# 2. 替换前端
ssh <staging_user>@<staging_host> \
    "docker cp /opt/micro-course/micro-course-admin/dist/.v1.22.0/. \
    micro-course-micro-course-admin-1:/usr/share/nginx/html/"

# 3. reload nginx
ssh <staging_user>@<staging_host> \
    "docker exec micro-course-micro-course-admin-1 nginx -s reload"

# 4. 验证首页
curl -s http://staging-host:8088/ | head -5
```

### 2.3 检查 staging 启动日志

```bash
# 后端启动日志（最近 2 分钟）
ssh <staging_user>@<staging_host> \
    "docker logs micro-course-micro-course-api-1 --since=2m" \
    | grep -E "Started|ERROR|Exception" | head -20

# 前端 reload 日志
ssh <staging_user>@<staging_host> \
    "docker logs micro-course-micro-course-admin-1 --since=1m" \
    | tail -10
```

## 5. 5 分钟监控

部署完成后保持监控 5 分钟：

```bash
# 1. 健康检查（连续 5 次）
for i in 1 2 3 4 5; do
  curl -s http://staging-host:8080/actuator/health
  sleep 60
done

# 2. 后端错误日志（最近 5 分钟）
ssh <staging_user>@<staging_host> \
    "docker logs micro-course-micro-course-api-1 --since=5m" \
    | grep -iE "error|exception" | head -20

# 3. 数据库慢查询
ssh <staging_user>@<staging_host> \
    "docker exec micro-course-postgres-1 psql -U microcourse_user micro_course -c \
    \"SELECT pid, query, state FROM pg_stat_activity WHERE state='active' AND query_start < now() - interval '5 seconds';\""

# 4. 容器资源使用
ssh <staging_user>@<staging_host> \
    "docker stats --no-stream micro-course-micro-course-api-1 micro-course-micro-course-admin-1"
```

### 监控告警触发条件（任一发生则立即回滚）

| 指标 | 阈值 | 行动 |
|------|------|------|
| `/actuator/health` 非 UP | 连续 2 次 | 立即回滚 |
| 后端 ERROR 日志 > 10 条/分钟 | 任意 1 分钟 | 立即回滚 |
| 数据库 active 查询 > 20 | 持续 1 分钟 | 立即回滚 |
| 容器内存 > 80% | 持续 2 分钟 | 排查是否回滚 |
| CPU > 90% | 持续 2 分钟 | 排查是否回滚 |
