## 五、架构风险提醒

### 视频流媒体（最大技术风险）

```
10,000 学生同时在线 → 视频并发读需求
学校服务器带宽是否够？ → 需提前测试
FFmpeg 转码队列管理 → 上传高峰期的队列积压
HLS 分片存储 → 大量 .ts 小文件 IO 压力

建议方案：
  - CDN 加速（必须）
  - 视频文件放在独立存储卷
  - 转码用消息队列异步处理
  - 提前进行带宽压测
```

### 数据库热点

| 热点表 | 风险 | 对策 |
|--------|------|------|
| video_progress | 高频写入（每5-10秒写一次） | 先写 Redis 再批量写 DB |
| exercise_attempts | 考试截止时间并发写入 | 队列削峰 |
| courses | 高并发读取 | Redis 缓存 + 缓存击穿保护 |

### 安全基线

| 项目 | 要求 |
|------|------|
| HTTPS | TLS 1.3 强制 |
| 密码 | bcrypt 加密 |
| JWT | Access 2h + Refresh 7d |
| 视频防盗链 | URL 签名 + 时间戳 + Referer |
| 隐私合规 | 符合《个人信息保护法》 |
