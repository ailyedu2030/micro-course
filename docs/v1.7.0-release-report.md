## 八、风险与已知限制

### 已记录 (非阻塞)
1. **P0-NEW-1 视频路径可枚举**: UUID 不可枚举, 但 URL 永久有效 (业界标准 trade-off)

## 九、灰度 2 周计划

### 阶段 1 (Day 1-3): 内部种子用户 (10 人)
- 选 10 个真实学生账号
- 监控关键指标: 选课成功率, 退课率, 通知到达率, 错误率

### 阶段 2 (Day 4-10): 部门级 (100 人)
- 推广到计算机系 + 商学院
- 监控: 并发选课是否触发候补, 通知是否 100% 送达

### 阶段 3 (Day 11-14): 院系级 (500 人)
- 推广到所有院系
- 监控: 视频上传 2GB 是否触发磁盘告警, 通知异步队列是否堆积

### 回滚条件
- 选课错误率 > 5% → 立即 `gray-release.sh rollback`
- 通知送达率 < 95% → 暂停推广
- 磁盘使用 > 80% → 限制上传

### 灰度脚本使用
```bash
bash scripts/gray-release.sh add seed-user-1
bash scripts/gray-release.sh add seed-user-2
...
bash scripts/gray-release.sh list
bash scripts/gray-release.sh roll-out  # 全量
bash scripts/gray-release.sh roll-back  # 回滚
```
