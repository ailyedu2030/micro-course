## 修复清单

### BUG-001 [P1-C] — SecurityConfig {id} 路径模式不匹配

| 属性 | 值 |
|------|-----|
| **状态** | ✅ **已修复** |
| **根因** | `MvcRequestMatcher` 不识别 Spring Web 的 `{id}` 路径变量语法，导致无认证请求匹配不到 `permitAll`，直接落入 `.anyRequest().authenticated()` 过滤链 |
| **修复文件** | `micro-course-api/src/main/java/com/microcourse/config/SecurityConfig.java:122-125` |
| **修改内容** | 将 3 条 `requestMatchers` 中的 `{id}` 替换为 Ant 通配符 `*`：<br>`/api/micro-specialties/*` → permitAll<br>`/api/micro-specialties/*/courses` → permitAll<br>`/api/micro-specialties/*/teachers` → permitAll |
| **验证方法** | `curl http://localhost:8080/api/micro-specialties/1` → HTTP 200 + 详情数据 |
| **防止再发** | SecurityConfig 中所有路径变量应使用 Ant 通配符 `*` 或 `**`，而非 `{name}` 语法 |

### BUG-002 [P1-C] — 学生申请修读 APPROVED 微专业提示"已结束"

| 属性 | 值 |
|------|-----|
| **状态** | ✅ **已修复** |
| **根因** | `MicroSpecialtyEnrollmentServiceImpl.apply()` 对非 `RECRUITING` 状态统一提示"已结束"，未区分"未开放招生"（DRAFT/PENDING_REVIEW/APPROVED）和"已结束"（COMPLETED/ARCHIVED/CANCELLED） |
| **修复文件** | `micro-course-api/src/main/java/com/microcourse/service/impl/MicroSpecialtyEnrollmentServiceImpl.java:111-123` |
| **修改内容** | 将单一状态检查拆分为两种场景：<br>• DRAFT/PENDING_REVIEW/APPROVED → "微专业当前未在招生期"<br>• COMPLETED/ARCHIVED/CANCELLED → "微专业已结束，无法操作" |
| **验证方法** | 对不同状态的微专业调用申请 API，检查返回消息准确性 |

### BUG-003 [P1-I] — 申报提交字段名不匹配

| 属性 | 值 |
|------|-----|
| **状态** | ✅ **已修复** |
| **根因** | 前端 JSON 请求体使用 `offerDepartmentIds`（复数）作为字段名，但后端 `MicroSpecialtyProposalRequest` 使用 `offerDepartmentId`（单数），Jackson 反序列化时无法匹配字段名，导致 `@NotNull` 校验触发"开课学院不能为空" |
| **修复文件** | `micro-course-api/src/main/java/com/microcourse/dto/microSpecialty/MicroSpecialtyProposalRequest.java:13-14` |
| **修改内容** | 在 `offerDepartmentId` 字段上增加 `@JsonProperty("offerDepartmentIds")` 注解，使 Jackson 能从 JSON 的 `offerDepartmentIds` 键反序列化到此字段 |
| **验证方法** | 前端 POST `{"offerDepartmentIds": 1, ...}` 不再触发校验错误 |
| **防止再发** | 前后端 API 契约文档中应标准化字段命名，避免单复数不一致 |

### BUG-004 [P1-I] — 重复操作/状态机错误码不一致

| 属性 | 值 |
|------|-----|
| **状态** | ✅ **已修复** |
| **根因** | Service 层多处使用 `MS_TERMINAL_STATUS(17019)` 作为终态错误码，与其他状态错误使用的 `MS_STATUS_INVALID(17003)` 不一致，导致前端无法统一处理"状态不允许"错误场景 |
| **修复文件** | 4 个文件共 14 处：<br>• `MicroSpecialtyAdminServiceImpl.java:98` — `checkNotTerminal()`<br>• `MicroSpecialtyFeaturedServiceImpl.java` — 6 处终态检查<br>• `MicroSpecialtyEnrollmentServiceImpl.java` — 3 处终态检查<br>• `MicroSpecialtyInviteServiceImpl.java` — 2 处终态检查 |
| **修改内容** | 将所有 `MS_TERMINAL_STATUS(17019)` 替换为 `MS_STATUS_INVALID(17003)`，携带定制消息"微专业已处于终态，无法操作" |
| **验证方法** | 对已归档/已取消微专业执行任何操作 → HTTP 400 + errorCode 17003 |
| **防止再发** | Status 相关错误统一使用 `MS_STATUS_INVALID(17003)`；新增枚举错误码需在 Code Review 中确认不与现有重复 |

### BUG-005 [P1-C] — set-gold-featured 返回 500 服务器内部错误

| 属性 | 值 |
|------|-----|
| **状态** | ✅ **已修复** |
| **根因** | `msRepository.acquireGoldFeaturedLock()` 调用 `SELECT pg_advisory_xact_lock(42)`。当 PG advisory lock 执行遇到异常时（如 MyBatisSystemException），抛出非 `BusinessException` 类型的异常，落入 `GlobalExceptionHandler.handleGeneric()` 返回 500 "服务器内部错误" |
| **修复文件** | `micro-course-api/src/main/java/com/microcourse/service/impl/MicroSpecialtyFeaturedServiceImpl.java:221-229` |
| **修改内容** | 在 `setGoldFeatured()` 中将 `acquireGoldFeaturedLock()` 调用包裹在 try-catch 中：<br>• 捕获 `Exception` 并记录错误日志<br>• 抛出 `BusinessException(ErrorCode.SERVICE_UNAVAILABLE, "系统繁忙，请稍后重试")` → 返回 503 友好错误 |
| **验证方法** | 并发调用 `set-gold-featured` → 一个成功，其他返回 503 + "系统繁忙，请稍后重试" |
| **防止再发** | 全局搜索所有 `@Select` 调用的 void 方法，确认其异常已被妥善包装；新增 DB 原生函数调用需考虑异常处理 |

### BUG-006 [P2] — unset-featured 守卫偏严

| 属性 | 值 |
|------|-----|
| **状态** | ✅ **已修复** |
| **根因** | `unsetFeatured()` 使用 `canTransitionTo(NONE)` 状态机守卫，该守卫仅允许 `APPROVED→NONE` 转换，不允许在 `PENDING` 或 `REJECTED` 状态下取消置顶 |
| **修复文件** | `micro-course-api/src/main/java/com/microcourse/service/impl/MicroSpecialtyFeaturedServiceImpl.java:197-212` |
| **修改内容** | 1. 守卫条件从 `canTransitionTo(NONE)` 改为 `featuredStatus == null || "NONE"` — 任何非 NONE 状态均可取消置顶<br>2. Update 条件从 `.eq(featuredStatus, "APPROVED")` 改为 `.in(featuredStatus, "PENDING", "APPROVED", "REJECTED")`<br>3. 错误消息从"仅已通过状态可取消置顶"改为"当前未设置置顶，无需取消" |
| **验证方法** | 对 PENDING/APPROVED/REJECTED 状态微专业取消置顶 → 均成功（200） |

