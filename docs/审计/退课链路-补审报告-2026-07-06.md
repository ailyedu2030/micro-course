## 退课状态机验证

```
学生退课流程（ENR-005）:
  APPROVED ──cancel──→ CANCELLED
    ① student_count -1 ← atomicDecrementStudentCount()
    ② WAITLIST → APPROVED ← promoteFirstWaitlistToEnrolled()
    ③ PAID Order → 退款 ← orderService.refund()
    ④ enrollment_histories 审计 ← recordHistory()
    ⑤ 通知学生 + 教师 ← ENROLLMENT_DROPPED ✅ F1+F2
```

**状态机约束**：
- APPROVED → CANCELLED ✅ 允许（白名单）
- COMPLETED → CANCELLED ❌ 被进度检查阻止
- 已取消选课 → CANCELLED ❌ `canTransitionTo()` 拒绝
- 终态 DROPPED/COMPLETED/REJECTED → CANCELLED ❌ 白名单拒绝
