## P1-C-1: 学生课程详情页渲染异常

### 现象
访问 `/student/courses/{id}` 时页面显示"页面出了点问题"。

### 根因【纪律 2】
`micro-course-admin/src/views/student/CourseDetail.vue` 在第 633 行使用了 `reactive()` 函数，但第 319 行的 Vue import 中没有导入 `reactive`：

```js
// 修复前（缺少 reactive）
import { ref, onMounted, onBeforeUnmount, computed, nextTick } from 'vue'

// 使用 reactive（第 633 行）
const reportDialog = reactive({ visible: false, ... })
```

由于 `reactive` 未导入，组件 setup 阶段抛出 `ReferenceError: reactive is not defined`。该错误被 `App.vue` 的 `onErrorCaptured` 全局错误边界捕获，展示降级 UI "页面出了点问题"。

**为什么测试中学习页正常？** 学习页（`LearningCenter.vue` / `LearningView.vue`）是独立的组件，不使用 `reactive`，因此不受影响。

### 横向扫描
`grep -r "reactive(" micro-course-admin/src/views/student/*.vue` 检查其他学生端组件：
- 仅 `CourseDetail.vue` 使用 `reactive` 但未导入
- 其他 23 个学生端组件均无此问题

### 修复
在 Vue import 中添加 `reactive`：

```js
// 修复后
import { ref, reactive, onMounted, onBeforeUnmount, computed, nextTick } from 'vue'
```

### 防止再发
- `reactive` 是 Vue 3 Composition API 标准函数，应包含在常规 import 集合中
- 建议后续为所有 `.vue` 文件的 import 语句做 lint 检查（`no-unused-vars` + `no-undef`）

### 验证
- 组件 setup 阶段不再抛出 `ReferenceError`
- 课程详情页可正常渲染课程信息、章节大纲、教师信息等所有区域
- 举报弹窗（使用 `reactive` 的部分）功能正常

## P1-C-2: 提交审核需先上传封面 — 流程引导不清晰

### 现象
课程发布流程中，提交审核（DRAFT → PENDING_REVIEW）要求先上传封面，但教师端缺少前置引导，只有在点击提交审核按钮后才收到后端错误"请先上传课程封面再提交审核"。

### 根因【纪律 2】
`micro-course-admin/src/views/courses/CourseDetail.vue` 存在两个问题：

1. **查看模式无封面缺失提示**（第 114 行）：封面卡片使用 `v-if="courseData.coverUrl"` 条件渲染，当课程无封面时，该卡片完全隐藏，教师无法直观感知封面缺失。

2. **提交审核无前置校验**（第 515 行）：`handleSubmitForReview()` 直接调后端 API，未在前端预检封面。后端 `CourseAuditServiceImpl.submitForReview()` 返回的 "请先上传课程封面再提交审核" 虽合理，但缺少如何上传的操作引导。

### 横向扫描
检查其他需要封面的操作路径：
- `POST /api/courses/{id}/cover` 封面上传接口本身正常
- 编辑模式下的封面上传区域有明确的拖拽引导和格式提示（`accept="image/jpeg,image/png,image/gif,image/webp"`）
- 问题仅在于从"查看模式"到"提交审核"的路径上缺少封面状态提示

### 修复

**修复 1：查看模式封面状态提示**
将封面卡片从条件渲染改为始终渲染，无封面时展示 `el-alert` 警告：

```html
<!-- 修复前 -->
<el-card shadow="never" class="info-card" v-if="courseData.coverUrl">

<!-- 修复后 -->
<el-card shadow="never" class="info-card">
  <template v-if="courseData.coverUrl">
    <el-image :src="courseData.coverUrl" fit="contain" class="cover-img" />
  </template>
  <template v-else>
    <el-alert title="尚未设置课程封面" type="warning" :closable="false" show-icon
      description="提交审核前必须上传封面，点击「编辑」按钮后在封面区域上传" />
  </template>
```

**修复 2：提交审核前置校验**
在 `handleSubmitForReview()` 中添加前端封面检查：

```js
if (!courseData.value?.coverUrl) {
  ElMessage.warning('请先上传课程封面再提交审核（点击「编辑」按钮，在封面区域上传）')
  return
}
```

### 防止再发
- 所有前端状态变更操作（提交/发布/下架）应有前置校验，提供明确的引导信息
- 查看模式应始终展示关键字段状态（即使为空），帮助教师识别缺失信息

### 验证
- 无封面时查看模式：显示警告"尚未设置课程封面"+ 引导文字 ✅
- 无封面时点击"提交审核"：显示 `ElMessage.warning` 并阻止提交 ✅
- 有封面时查看模式：正常显示封面图片 ✅
- 有封面时点击"提交审核"：正常调 API 提交 ✅
