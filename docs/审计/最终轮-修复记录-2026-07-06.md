## Phase 5 缺陷

### [P1-C] BUG-001: 选课通知课程名称 null

**根因**: `CourseRepository.selectByIdForUpdate()` 的 SQL 查询未包含 `title` 列，导致 `lockedCourse.get("title")` 返回 null。

**修复文件**:
- `micro-course-api/src/main/java/com/microcourse/repository/CourseRepository.java:41`
  - 在 `SELECT` 中添加 `title` 列

**验证**: 选课后通知中的课程名将从 `lockedCourse` 中正确获取。

---

### [P1-I] BUG-002: GET /api/chapters 缺 courseId 返回 200

**根因**: `courseId` 参数为 `@RequestParam(required = false)`，null 时直接返回空列表。

**修复文件**:
- `micro-course-api/src/main/java/com/microcourse/controller/CourseChapterController.java:34-36`
  - 添加 `if (courseId == null) throw BusinessException(BAD_REQUEST_PARAM, "courseId 不能为空")`

**验证**: 缺 courseId 时返回 HTTP 400 + code=9005。

---

### [P1-I] BUG-003: PUT /api/auth/me 空 body 返回 200

**根因**: `AuthServiceImpl.updateProfile()` 未校验至少需要一个字段。

**修复文件**:
- `micro-course-api/src/main/java/com/microcourse/service/impl/AuthServiceImpl.java:608-618`
  - 添加空 body 校验：`realName/email/phone/gender` 全为 null 时抛出 9005

**验证**: `PUT /api/auth/me -d '{}'` 返回 code=9005 "至少需要更新一个字段"。

---

### [P2] BUG-004: GET /api/courses?size=99999 应 clamp

**根因**: `@Range(min = 1, max = 10000)` 注解直接拒绝超大 size，而非 clamp。

**修复文件**:
- `micro-course-api/src/main/java/com/microcourse/controller/CourseController.java:41,60,66-68`
  - 添加 `MAX_PAGE_SIZE = 200` 常量
  - 移除 `@Range` 注解（改为 `@PositiveOrZero`）
  - 添加 clamp 逻辑：`size < 1 ? 1 : size > MAX_PAGE_SIZE ? MAX_PAGE_SIZE : size`

**验证**: `GET /api/courses?size=99999` 正常返回 200，size 被 clamp 为 200。

## Phase 11 缺陷

### [P1-C-01] 后端热拔插不完整

**根因**: `InteractivePluginAutoConfig` 有 `@ConditionalOnProperty`，但 3 个 Controller 和 4 个 Service 缺少条件注解，被 `@SpringBootApplication` 隐式扫描注册。

**修复文件**（每文件添加 `@ConditionalOnProperty`）:
1. `SlideController.java:32` — 类级别添加
2. `NarrationController.java:14` — 类级别添加
3. `TtsController.java:16` — 类级别添加
4. `SlideServiceImpl.java:50` — 类级别添加
5. `NarrationServiceImpl.java:40` — 类级别添加
6. `TtsServiceImpl.java:49` — 类级别添加
7. `SlideRenderService.java:38` — 类级别添加

**统一注解**: `@ConditionalOnProperty(value = "plugin.interactive.enabled", havingValue = "true", matchIfMissing = true)`

**验证**: `plugin.interactive.enabled=false` 时所有 plugin Bean 不注册。

---

### [P1-I-01] 批量 AI/TTS 异步任务无前端可见错误反馈

**根因**: `generateAll()` 异步方法失败时仅记日志，前端轮询无法感知。

**修复文件**:
- `NarrationServiceImpl.java:155-159` — DeepSeek Key 缺失时标记 DB 状态
- `NarrationServiceImpl.java:198-201` — DeepSeek API 调用失败时标记 DB 状态
- `NarrationServiceImpl.java` — 新增 `markAllPagesError()` 方法
- `TtsServiceImpl.java:361-365` — TTS 后端不可用时提前返回并标记 DB 状态
- `TtsServiceImpl.java` — 新增 `markAllPagesError()` 方法

**验证**: 批量生成失败后 DB 中各页面的 `narration_status` 被标记为初始/可编辑状态。

---

### [P1-I-02] 文件上传校验层级不一致

**根因**: Controller 层魔数校验允许 JPEG/PNG，Service 层只允许 PPTX，错误消息不一致。

**修复文件**:
- `SlideController.java:212-236` — 移除 JPEG/PNG 魔数校验分支，仅保留 ZIP(PPTX) 校验
- 统一错误消息为 "不支持的课件格式（仅支持 PPTX 格式）"

**验证**: JPEG/PNG 文件上传返回 code=16008，与 PPTX-only 策略一致。

---

### [P2-01] 不存在页面的图片端点返回 200 而非 404

**根因**: `SlideServiceImpl.getPageImage()` 在 `getPage()` 抛出 BusinessException 时返回占位图。

**修复文件**:
- `SlideServiceImpl.java:210-232` — 页面不存在时重抛异常（404），slideId 为 null 时也抛异常
- 仅对文件读取失败（IO 异常）保留 fallback 占位图

**验证**: `GET /slides/pages/999/image` 返回 code=16004（SLIDE_PAGE_NOT_FOUND）。
