## 发现的问题

### P1-C（客户可感知）

#### [BUG-001] 选课通知中课程名称为 null
- **端点**: `POST /api/enrollments`
- **描述**: 选课后生成的系统通知内容为"您已成功选课《null》，开始学习吧！"，课程名称为null
- **复现**: 以 student5 身份选课 courseId=3（数据库系统原理），查看 notifications 表中 id=71 的记录
- **根因**: enrollment 流程创建通知时未正确获取课程名称
- **截图**:
  ```json
  {"id":71,"content":"您已成功选课《null》，开始学习吧！","relatedId":3}
  ```
- **修复建议**: 选课流程中在创建通知前查询 course name，保证 content 中的课程名正确

### P1-I（内部可见）

#### [BUG-002] `GET /api/chapters` 缺少必填参数 `courseId` 时返回 200
- **端点**: `GET /api/chapters`
- **描述**: 不传 courseId 参数时返回 200（data 为空列表），应返回 9005 参数校验错误
- **复现**: `curl http://localhost:8080/api/chapters`
- **风险**: 低，前端不会无courseId调用此接口，但API不够严谨
- **修复建议**: 对 courseId 添加 `@RequestParam(required = true)` 或 `@NotNull` 校验

#### [BUG-003] `PUT /api/auth/me` 空 body 时返回 200
- **端点**: `PUT /api/auth/me`
- **描述**: 不传任何字段（`{}`）时仍返回 200，没有任何更新，但应返回 9005 告知至少需要一个字段
- **复现**: `curl -X PUT -H "Content-Type: application/json" -d '{}' /api/auth/me`
- **风险**: 低，前端不会发送空 body
- **修复建议**: 在 Service 层校验至少有一个非空字段需要更新

### P2（代码整洁/体验优化）

#### [BUG-004] `GET /api/courses?page=0&size=99999` 返回 9005 而非 clamp 到最大值
- **端点**: `GET /api/courses`
- **描述**: 当 size 参数过大（如 99999）时返回 9005 错误，建议 clamp 到系统最大值（如 100 或 200）
- **风险**: 低
