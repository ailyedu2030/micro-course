## 关键发现

### 1. 权限矩阵正确性验证
所有端点的 `@PreAuthorize` 注解与权限矩阵 v2.0 对齐：
- `isAuthenticated()` 端点: 无 token → 401 ✅
- `hasAnyRole('ADMIN','ACADEMIC')` 端点: TEACHER → 403 ✅
- `hasRole('ADMIN')` 端点: TEACHER → 403 ✅  
- `hasAnyRole('TEACHER','ADMIN')` 端点: STUDENT → 403 ✅
- `hasRole('STUDENT')` 端点: TEACHER → 403 ✅
- `hasAnyRole('ADMIN','ACADEMIC')` 端点: 写操作不允许 TEACHER ✅

### 2. 端点可用性
- 基础数据 CRUD 全部正常工作
- 课时创建/排序/更新/删除全部通过
- 视频书签鉴权正确
- 成绩管理完整 CRUD + 批改 + 评阅流程通过
- 操作日志分页+多维筛选通过

### 3. Redis 状态污染注意事项
- 测试依赖共享 Redis 实例 (localhost:6380)，前序测试的登录锁定会跨测试类污染
- `resetLoginState()` 在 `@BeforeEach` 中部分覆盖，但对 `JWT 黑名单` 清理不彻底
- **规避方案**: 本测试在 `@BeforeEach` 中防御性重置 admin 密码哈希
- 首次运行前需 `redis-cli -p 6380 FLUSHDB` 清除遗留状态
