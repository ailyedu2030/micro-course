### P1-C-02: DELETE 返回200但不生效

| 项目 | 内容 |
|------|------|
| **根因** | `CourseAdminServiceImpl.delete()` 仅将课程 status 设为 CLOSED，未设置 `deletedAt`。Course 实体存在 `@TableLogic(value = "NULL", delval = "now()")` 但 delete 方法使用 `LambdaUpdateWrapper`（非 `deleteById`），导致 `@TableLogic` 未被触发。CLOSED 课程仍可通过 `selectById` 查询到。 |
| **修复** | 在 delete 方法的 `LambdaUpdateWrapper` 中增加 `.set(Course::getDeletedAt, LocalDateTime.now())` |
| **文件** | `CourseAdminServiceImpl.java:281` |
| **验证** | 软删除后 `@TableLogic` 自动在 `selectById`/`selectList` 中加入 `WHERE deleted_at IS NULL`，GET 返回 404(COURSE_NOT_FOUND)，符合"删除后不可见"预期 |
