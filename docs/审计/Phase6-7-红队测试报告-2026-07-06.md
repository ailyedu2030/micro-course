## 三、缺陷清单

### P0 级（数据安全 / 核心功能不可用）— 0 个

未发现 P0 级别缺陷。

### P1-C 级（客户可感知）— 3 个

#### 🔴 P1-C-01: 课程复制产生已发布副本

| 属性 | 值 |
|------|-----|
| **API** | `POST /api/courses/{id}/copy` |
| **表现** | 复制原课程(Java程序设计基础, status=4)后，副本自动获得 status=5 (PUBLISHED) 而非 DRAFT(0) |
| **影响** | 副本直接可公开访问，教师无法编辑，可能暴露未准备就绪的内容 |
| **根因** | copy 逻辑未将新课程状态重置为 DRAFT |
| **位置** | `CourseService.copy()` 或相关实现 |
| **证据** | 副本ID=63, status=5; 原ID=1, status=4 |

#### 🔴 P1-C-02: DELETE 返回 200 但资源未被删除

| 属性 | 值 |
|------|-----|
| **API** | `DELETE /api/courses/{id}` |
| **表现** | DELETE 返回 `{code:200, message:"ok"}` 后，`GET /api/courses/{id}` 仍能查到完整课程数据，状态仍为 5 |
| **影响** | 教师看到"删除成功"但数据仍然存在，产生信任危机 |
| **根因** | DELETE 可能是软删除但未变更状态/可见性，或 SQL 未实际执行 |
| **证据** | 课程 63/64 delete→200 后 GET 仍返回 status=5 |

#### 🔴 P1-C-03: 课程复制不含章节

| 属性 | 值 |
|------|-----|
| **API** | `POST /api/courses/{id}/copy` |
| **表现** | 副本课程无任何章节（原课程有3个章节） |
| **影响** | 教师需要重新手动创建所有章节和视频，复制功能形同虚设 |
| **根因** | copy 逻辑只复制了课程主表记录，未递归复制 chapters/lessons/videos |
| **证据** | 原ID=1 → 3 chapters; 副本ID=63 → 0 chapters |

### P1-I 级（内部仅见）— 3 个

#### 🟡 P1-I-01: 操作日志响应格式不一致

| 属性 | 值 |
|------|-----|
| **API** | `GET /api/operation-logs` |
| **表现** | 部分日志返回 `{action, module, detail:{method,path,status}, duration, ip}` 格式; 另一部分日志返回 `{action, module, userId, targetType, targetId, createdAt}` 格式，无 duration |
| **影响** | 前端渲染日志列表时可能出现空字段 |
| **根因** | 两种日志来源(AOP拦截 vs 手动记录)使用了不同的 VO |
| **证据** | log ID=2763 有 detail JSON, log ID=2762 无 detail |

#### 🟡 P1-I-02: 课程状态变更返回通用错误码

| 属性 | 值 |
|------|-----|
| **API** | `POST /api/courses/{id}/approve`, `PUT /api/courses/{id}/status` |
| **表现** | 返回 `{code:6005, message:"审核通过失败，请检查课程状态"}` — 未具体说明当前状态和期望状态 |
| **影响** | 开发和排障时需要额外查询课程状态 |
| **根因** | ErrorCode 定义过于通用，未区分具体状态机转换条件 |

#### 🟡 P1-I-03: 教学班添加学生后查询为空

| 属性 | 值 |
|------|-----|
| **API** | `POST /api/teaching-classes/{id}/students` → `GET /api/teaching-classes/{id}/students` |
| **表现** | 添加学生 API 返回 200 后，立即查询学生列表返回 0 |
| **影响** | 疑似添加操作未持久化或查询范畴有误 |
| **位置** | `TeachingClassController.addStudent()` 或对应 Service |
