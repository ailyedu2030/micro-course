| 6.10 | `/api/offline-sessions/{id}/attendance/{recordId}` | PUT | TEACHER/ADMIN | ✅ 200 | 更新签到状态 |

### 6.2 签到流程验证

```
教师: 创建线下活动 (chapter=76, date=2026-07-10) ✓
教师: manual-checkin student 7 → 200 ✓
教师: GET attendance → [student=陈小明, status=PRESENT, checkinTime=...] ✓
学生: GET my-attendance → [status=PRESENT] ✓
学生: POST checkin → 9005「签到时间不在活动日期范围内」(sessionDate=07-10, today=07-06) ✓
教师: DELETE session → 200 ✓
```

### 6.3 注意事项

- `POST /api/offline-sessions/{chapterId}/chapters` 使用 `{chapterId}` 作为离线章节 ID，需要是 `chapterType=OFFLINE` 的章节
- 必填字段：`sessionDate` (LocalDate), `startTime` (LocalTime), `endTime` (LocalTime), `sortOrder` (Integer)
- 学生签到有时间窗口校验（只能在活动日期范围内签到）
