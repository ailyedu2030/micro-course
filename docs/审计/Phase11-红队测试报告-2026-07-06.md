## 四、缺陷报告

### P0 级：无

---

### P1-C 级：1 项

#### [P1-C-01] 后端热拔插不完整 — 控制器缺少 @ConditionalOnProperty

**严重性**：P1-C（客户可感知）
**位置**：
- `SlideController.java:29-31` — 缺少 `@ConditionalOnProperty`
- `NarrationController.java:11-13` — 缺少 `@ConditionalOnProperty`
- `TtsController.java:14-16` — 缺少 `@ConditionalOnProperty`
- `SlideServiceImpl.java:48-49` — 缺少 `@ConditionalOnProperty`
- `NarrationServiceImpl.java:38-39` — 缺少 `@ConditionalOnProperty`
- `TtsServiceImpl.java:47-48` — 缺少 `@ConditionalOnProperty`
- `SlideRenderService.java:35-36` — 缺少 `@ConditionalOnProperty`

**描述**：只有 `InteractivePluginAutoConfig` 和 `NarrationSettingController` 带有 `@ConditionalOnProperty(name = "plugin.interactive.enabled", havingValue = "true")`。其余 3 个 Controller 和 4 个 Service 无此注解。当 `plugin.interactive.enabled=false` 时：
1. 自动配置不加载 → `slideRenderExecutor` Bean 缺失
2. 但所有 Controller 仍然注册（被 `@SpringBootApplication` 隐式扫描）
3. 7 个 API 入口点仍然可访问
4. 调用时因 Executor Bean 缺失 → 启动失败或运行时异常

**建议修复**：所有 Controller 和 Service 添加 `@ConditionalOnProperty`，或使用 `@ComponentScan` 配合 excludeFilters 确保整个 package 被条件控制。

**对照**：`NarrationSettingController.java:14` 已有正确实现可作为模板。

---

### P1-I 级：2 项

#### [P1-I-01] 批量 AI/TTS 异步任务无前端可见错误反馈

**位置**：
- `NarrationServiceImpl.java:153-237` — `generateAll()` 异步执行
- `TtsServiceImpl.java:358-378` — `generateAll()` 异步执行

**描述**：批量 AI 生成/批量 TTS 生成的异步任务在执行失败后（如 DeepSeek API Key 未配置），API 立即返回 code 200，但实际处理静默失败，仅记日志。前端轮询 `narrationStatus` 不会收到错误提示。用户无法区分"正在处理中"和"已静默失败"。

**建议修复**：添加 `CompletableFuture` 返回，或使用 WebSocket/SSE 推送异步任务状态，或设置 DB 字段记录错误信息。

**对客户影响**：教师点击"批量生成"后可能一直在等待，看不到失败原因。

#### [P1-I-02] 文件上传校验层级不一致

**位置**：
- `SlideController.java:67` — `validateSlideFileMagic()` 允许 ZIP/JPEG/PNG 魔数
- `SlideServiceImpl.java:93-94` — 仅检查 `.pptx` 扩展名

**描述**：Controller 层魔数校验允许 JPEG/PNG，错误消息显示"仅支持 PPTX/JPEG/PNG 格式"；Service 层文件名校验仅允许 `.pptx`。两层校验标准不一致，且错误消息不同（Controller: "仅支持 PPTX/JPEG/PNG 格式"，Service: "不支持 PPT 格式，请上传 .pptx 文件"）

**建议修复**：统一为仅允许 PPTX，去除 JPEG/PNG 魔数校验分支，使错误消息一致。

---

### P2 级：2 项

#### [P2-01] 不存在页面的图片端点返回 200 而非 404

**位置**：`SlideServiceImpl.java:210-232` — `getPageImage()`

**描述**：请求不存在的页面图片（`GET /slides/pages/999/image`）返回 HTTP 200 和一个占位图，而非 404 或错误码。虽然 fallback 机制对未知页面号的播放器体验有容错价值，但不返回 404 可能被爬虫/扫描器误认为资源存在。

**建议修复**：对确定不存在的页号返回 404，仅对渲染中的页使用 fallback 占位。

#### [P2-02] 音频获取端点路径穿越防护未覆盖所有场景

**位置**：`TtsServiceImpl.java:381-395` — `getAudio()`

**描述**：`getAudio()` 实现了 `toRealPath()` + `startsWith` 路径穿越防护，但 `audioDuration` 计算使用 `estimateDuration(fileSize/2000)`，粗略估算不一定准确。
