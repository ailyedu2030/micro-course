## 第 0 部分 · 审计方法与编号规则

### 0.1 用户操作最小颗粒度定义

**最小审计单元** = 用户在 UI 上的**单次离散操作**，包含以下任一类型：

| 类型 | 例子 | 是否独立编号 |
|------|------|:-----------:|
| 单次点击 | 点击"登录"按钮 | ✅ |
| 单次输入 | 用户名输入框 blur | ✅ |
| 单次切换 | 难度筛选 change | ✅ |
| 单次提交 | 表单提交 | ✅ |
| 单次回退 | 取消按钮 | ✅ |
| 单次中断 | 网络断开 | ✅ |
| 单次状态变更 | Switch 切换 | ✅ |
| 单次路由跳转 | Tab 切换 | ✅ |
| 单次 Dialog 弹出 | 点击"导出"打开 Dialog | ✅ |
| 单次校验失败 | blur 校验失败 | ✅ |

### 0.2 编号规则

- **统一前缀**：`OP-NNNN`（4 位顺序号，从 0001 开始）
- **域前缀**（仅标注用，便于分类）：AUTH/STU/TCH/ADM/ACA/BASE/CONT/NOTIF/APPROVE/ISLIDE/LOCAL
- **链路归属**：每条业务链路（S01-S32 / T01-T25 / A01-A30 / AC01-AC14）
- **三视角引用**：每个单元同时引用 BE 端点、FE 触发动作、DB 表/约束

### 0.3 与现有 2026-07-06 审计的关系

| 维度 | 现有 2026-07-06 审计 | 本次细粒度审计 |
|------|---------------------|---------------|
| 拆分粒度 | 101 条业务链路 | **318 个单步操作节点** |
| Agent 拆分 | 11 Agent 按业务域拆分 | **12 Agent 按离散单步拆分** |
| 单 Agent 持有 | 6-30 条业务链路 | **0 条完整链路 + ~26 个离散步骤** |
| 审查单元 | 整条业务链路 | **单次用户操作** |

**结论**：本次审计粒度是现有审计的 ~3 倍细化。
