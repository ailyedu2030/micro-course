# 重点摘要

## 🔴 P1-C — OP-0227：自审批漏洞
**文件**：`MicroSpecialtyProposalServiceImpl.java:288-331`
**问题**：`approveAndCreateSpecialty()` 方法**未校验审批人是否为申报人本人**。如果一个用户同时拥有 TEACHER 和 ACADEMIC 角色，可以自行批准自己的微专业申报，绕过正常审批流程。
**建议**：在方法开头增加 `if (reviewerId.equals(proposal.getProposerId()))` 校验拦截。

## ⚠️ P1-I — OP-0179：前端 LEAD 人数校验缺失
**文件**：`MicroSpecialtyManage.vue`（showOpen 计算属性）
**问题**：开课按钮仅在 status === 'APPROVED' 时启用，未检查 LEAD 团队人数 ≥ 2 的业务规则。用户点击后可能收到后端错误。
**建议**：从详情接口获取团队列表后计算 `leadCount >= 2` 条件，加到 `showOpen` 计算属性中。

## ⚠️ P1-I — OP-0023：路由重定向无提示
**文件**：`router/index.js:249`
**问题**：非 TEACHER 访问 requiresLead 路由时静默重定向，无任何提示。
**建议**：重定向前加 `ElMessage.warning('您没有权限访问该页面')`。

## ⚠️ P1-I — OP-0251：前后端权限粒度不一致
**文件**：`DepartmentList.vue:63` / `DepartmentController.java`
**问题**：前端用 `v-if="userRole !== 'ACADEMIC'"` 隐藏编辑按钮，但后端 `@PreAuthorize("hasAnyRole('ADMIN','ACADEMIC')")` 允许 ACADEMIC 编辑。
**建议**：统一权限策略。
