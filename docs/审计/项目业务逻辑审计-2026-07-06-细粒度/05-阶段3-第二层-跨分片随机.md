## 跨域联动场景 2：学生端选课 → 订单支付

**场景 ID**: CS-02
**涉及操作单元**:
- 单元 A: OP-0037 (Agent 1) — 点击"立即报名"（免费/付费多分支）
- 单元 B: OP-0052 (Agent 4) — 点击"确认支付"（批量结算）

### 单节点审查结论
- 单元 A：[P1-I] Agent 1 报告：OrderController 缺少 STUDENT 角色专用 `@PreAuthorize`，TEACHER 理论上可创建订单
- 单元 B：[P1-C] Agent 4 报告：`batchCreate()` 事务边界问题——单个失败导致全部回滚，且 pay 失败后仅日志记录，前端误判为全成功

### 跨域联动分析

#### 1. 数据流上下游完整性
- 上游输出：OP-0037 创建订单（PENDING 状态），通过 `POST /api/orders` 返回订单 VO
- 下游输入：OP-0052 读取购物车 store 中的课程列表，调用 `batchCreateOrders`
- **是否断裂**：[✗] — 关键断裂点：OP-0037 的"立即报名"与 OP-0052 的"购物车结算"是两条独立的**并行路径**——OP-0037 针对单课程立即报名（创建订单+支付），OP-0052 针对购物车批量结算。两条路径**没有交叉**，但共享相同的 `createOrder` 和 `pay` 后端方法。Agent 4 报告的 batchCreate 事务缺陷（P1-C）意味着：用户通过购物车批量结算时，若其中一门课程失败，整个事务回滚。但"立即报名"路径的单课程操作不受此影响。**功能性断裂不存在，但事务设计缺陷存在于批量路径**

#### 2. 异常处理跨域传播
- 上游异常处理："立即报名"失败 → `ElMessage.error` + 设置 `isEnrolled=false`
- 下游感知：批量结算部分失败 → 降级为逐一下单+支付，但仍可能全部失败
- **是否被忽略**：[✗] — 降级路径有兜底

#### 3. 背景条件跨域稳定性
- 上游假设的条件：课程需为 selectable 状态，价格需有非 REJECTED 的定价记录
- 下游依赖的条件：购物车 store 中的课程有效
- **是否被破坏**：[✗] — 条件独立验证，不受对方影响

#### 4. 责任边界是否明确
- 上游职责：免费→直接选课；付费→创建 PENDING 订单
- 下游职责：批量创建订单 + 批量支付 + 失败明细展示
- **是否清晰**：[✓]

### 联动缺陷判定

| 项目 | 内容 |
|------|------|
| **联动缺陷 ID** | LD-002 |
| **风险等级** | **P1-C** — 非典型联动断裂，而是两条并行业务路径共享后端逻辑导致的不一致 |
| **根因分类** | 责任边界盲区 |
| **冲突点** | OP-0037（单课程直接报名）和 OP-0052（批量结算）共享 `OrderServiceImpl.createOrder()` 方法。OP-0037 路径有前端 `isFreeForMe` 判断决定走免费/付费分支，但 OP-0052 路径的 Checkout.vue 不展示免费/付费标签——批量结算页面统一展示"确认支付"，如果用户购物车同时有免费和付费课程，免费课程也会被错误地引导进支付流程 |
| **影响范围** | 购物车同时含免费+付费课程时，用户可能为免费课程付费 |
| **修复建议** | `Checkout.vue` 购物车列表应展示每门课程的 isFreeForMe 标识，免费课程标注"免费"并跳过支付流程。`batchCreateOrders` 后端应区分 free/paid 课程：免费课程走免费选课路径，不创建订单 |
