## 一、背景

PR #15 (`fix/business-logic-audit-165-defects`) 包含 165 项业务逻辑审计修复，分支从 pre-PR-30 的 main 创建。

**P0 冲突发现 (2026-07-18)**：PR #30（v1.22.0）修改了 `OrderServiceImpl.java`（套餐发课修复）和 `MicroSpecialtyProposal.vue`（teacherId 占位修复），PR #15 从旧 main 分叉，直接合入会**覆盖 PR #30 的修复**。

**决策 (D2)**：拆分 PR #15，保留与 v1.22.0 不重叠的 ~120 项，关闭原 PR #15。

---

## 二、拆分原则

1. **绝不覆盖 v1.22.0 修复**（PR #30 / PR #32 / PR #33 已合入的内容优先保留）
2. **按模块拆分**：后端 service → 后端 controller → 前端 → 测试 → 配置
3. **保留变更**：不与 v1.22.0+ 冲突的 ~120 项审计修复
4. **丢弃变更**：与 v1.22.0+ 冲突/重复/已被更优方案替代的修复

## 五、执行步骤

1. checkout PR #15 分支
