## 五、决策交底与跟踪复盘机制

### 5.1 交底对象

- **直接交底**: owner 本人 (ailyedu2030)
- **间接交底**: 团队成员 (如有)，通过 README + 协作方案 (PR #36)
- **公开交底**: GitHub Discussions + 项目博客

### 5.2 交底内容

每个决策项的：
- 决策方向（推荐 vs 备选）
- 责任主体
- 完成节点
- 验收标准
- 风险与回退方案

### 5.3 跟踪复盘机制

| 时间点 | 动作 | 文档 |
|--------|------|------|
| 每日 17:00 | 进度同步 | `docs/daily-standup-2026-07-XX.md` |
| 每周五 16:00 | 周度复盘 | `docs/weekly-retro-2026-WXX.md` |
| 决策节点 | 验收检查 | `docs/decisions/checkpoints/` |
| 月度 | 总复盘 | `docs/monthly-retro-2026-XX.md` |

### 5.4 偏差升级路径

```
进度偏差 > 1 天    → 立即在 #standup 频道报警
进度偏差 > 3 天    → 启动应急 plan B
决策未达成         → 升级到 owner + AI 联合复盘
事故触发 (P0/P1)   → 立即停止 + 写事故复盘 + 24h 内复盘
```

### 5.5 复盘模板

每周末提交 `weekly-retro-YYYY-WXX.md`，包含：
1. **完成项** vs **计划项** — 偏差率
2. **决策执行情况** — D1-D8 各项状态
3. **新发现风险** — 是否有 P0/P1 新问题
4. **下周计划** — 重新调整的 D1-D8 节点
5. **教训** — 1-3 条学习
