# 错误复盘报告模板 · W33 治理

> **铁律**: 任何已修复错误 3 个工作日内输出根因分析报告.
> **依据**: 用户第 22 次授权铁律 - "建立错误复盘机制...避免同类问题重复出现".

---

## 复盘报告结构 (5 段 1 列表)

### 1. 错误基本信息
| 字段 | 值 |
|------|---|
| 错误 ID | ERR-YYYYMMDD-NNN |
| 错误等级 | P0 / P1 / P2 / P3 |
| 错误名称 | (简短) |
| 报告人 | 总工程师 |
| 报告时间 | YYYY-MM-DD HH:MM |
| 首次发生 | YYYY-MM-DD HH:MM |
| 修复时间 | YYYY-MM-DD HH:MM |
| 响应时长 | (vs SLA 1h/2h/7d/30d) |
| 修复时长 | (vs SLA 4h/24h/7d/30d) |

### 2. 错误现象
- **用户视角**: 用户看到的现象
- **业务影响**: 影响范围 (e.g. 100% 教师无法删除课件)
- **触发条件**: 复现步骤
- **错误日志**: 关键 stacktrace / log

### 3. 根因分析 (5 Why)
1. Why: 为什么发生？
2. Why: 为什么代码没拦截？
3. Why: 为什么测试没发现？
4. Why: 为什么监控没及时告警？
5. Why: 为什么流程没阻止？

### 4. 修复方案
- **临时方案**: 立即止血 (e.g. 重启 / 回滚 / 配置变更)
- **永久方案**: 代码修复 + 配置修改
- **回归测试**: 跑回归测试套件 (3 阶段全过)
- **回滚方案**: 5 分钟内可降级

### 5. 防范措施
| 层级 | 措施 | 责任 | 截止 |
|------|------|------|------|
| 流程 | e.g. 增加 PR 模板 | 总工程师 | W34 |
| 技术 | e.g. 单元测试覆盖该路径 | 总工程师 | W34 |
| 监控 | e.g. 新增告警规则 | 总工程师 | W34 |
| 文档 | e.g. 更新 README | 总工程师 | W34 |

### 6. 改进效果 (修复后 7 天跟踪)
- 同类错误 7 天内再次发生？ (是/否)
- 监控告警是否有效？ (是/否)
- 用户满意度变化？ (数值)
