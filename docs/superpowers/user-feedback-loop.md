# 用户反馈闭环机制 · 2026-07-20

> 由总工程师建立, 服务客户体验至上铁律.

## 一、反馈渠道 (5 路)

| 渠道 | 路径 | 响应 SLA |
|------|------|---------|
| **教师工单** | 教师面板 → 帮助 → 提交反馈 | 2 小时首次响应 |
| **管理员上报** | 管理员 → 用户管理 → 反馈列表 | 1 小时首次响应 |
| **生产监控告警** | Prometheus → AlertManager → Slack | 实时 (秒级) |
| **用户调研** | 月度 NPS 问卷 | 月度总结 |
| **GitHub Issues** | ailyedu2030/micro-course/issues | 4 工作日 |

## 二、问题定级

| 信号 | 级别 | 处理 |
|------|------|------|
| 5xx 比例 > 1% / 核心 API 不可用 / 登录失效 | **P0** | 15 min 启动, 1 h 修复 |
| 部分课件音频加载失败 / 单用户场景异常 | **P1** | 4 h 定位, 24 h 修复 |
| 样式偏移 / 文案错 / 加载延迟 1-3s | **P2** | 7 天内修复 |
| UI 优化 / 边缘场景 | **P3** | 30 天优化排期 |

## 三、修复闭环流程

```
反馈 → 总工程师接收 → 5 分钟内分类定级 → 进入 SLA 时钟 → 修复 → 
  → 多环境验证 (本地 + 预发布 + 生产) → 用户告知 → 根因报告 → 知识库归档
```
