## 6. 改进建议

### 6.1 P0 改进项（建议立即修）

**权限矩阵 §1.1 vs API 契约 v1.1 端点不一致**：

| 端点 | 权限矩阵 v2.0 | API 契约 v1.1 | 状态 |
|------|--------------|--------------|------|
| GET /api/auth/me | ✓ | §1.5 | ✅ |
| **PUT /api/auth/me** | ✓ | **缺** | ⚠️ |
| **PUT /api/auth/me/password** | ✓ | **缺** | ⚠️ |
| **POST /api/auth/me/avatar** | ✓ | **缺** | ⚠️ |

**冲突点**：权限矩阵定义 4 个 /me 端点，API 契约 v1.1 仅 1 个。

**建议处理**（总工程师决策待定）：
- A. API 契约 v1.1 补 3 个端点（与权限矩阵对齐）
- B. 权限矩阵 v2.1 删除 3 个端点（与 API 契约对齐）
- C. 写为「Phase 1.5 补」，暂不实现

### 6.2 P1 改进项

1. **SecurityConfig 缺详细路径配置**：当前 application.yml 没有 `spring.security.user.name/password` 等显式声明，但因为 Phase 2.5（Phase 1 G5）已物理删除 SecurityConfig，这是预期状态
2. **CSRF 状态未明示**：开发规范 v1.3 §7.2 未提 CSRF，但 Spring Security 6 默认启用——应在 SecurityConfig 重建时显式 `csrf().disable()`
