## 1. 问题

### 1.1 当前错误

教师创建"子资源"(视频/互动课件/线下课场次/试卷)时，有 4 个页面不让你选课程和章节：

| 页面 | 当前行为 | 错在哪 |
|------|---------|--------|
| `TeacherSlideOverview.vue` | "新增互动课程"弹窗只让你填 title/category/description,然后**自动创建新课程**(！)。它应该只是选课程→选章节→上传 PPT | 路径完全错了 |
| `SlideManage.vue` | 路由绑定 courseId,但表单里没有选章节的地方 | 上传 PPT 只能挂课程级,不能挂章节级 |
| `TeacherOfflineSessions.vue` | 路由绑定 chapterId,有面包屑标题但表单未显式显示"为《章节》添加场次" | 虽有章节上下文但不够明确 |
| `ExamList.vue` | 智能组卷表单没有课程/章节选择 | 试卷无法关联到具体章节 |

### 1.2 原因

核心问题是模型设计本身是正确的,但前端没对齐:

```
Course (课程)
  ├── Chapter (章节) — 关键类型字段: chapterType ∈ {VIDEO, INTERACTIVE, OFFLINE, EXERCISE}
  │     ├── [VIDEO] → videos (视频列表)
  │     ├── [INTERACTIVE] → slide_pages (互动课件)
  │     ├── [OFFLINE] → chapter_offline_sessions (线下课场次)
  │     └── [EXERCISE] → exercises/exams (练习/考试)
```

但前端长期缺失"按章节创建子资源"的 UX 模式,造成了**路径断裂**。

### 1.3 影响

- 教师不能"为第 3 章上传一个互动课件 PPT"
- 不能"为第 5 章添加一场线下课"
- 不能"为第 7 章创建一个练习试卷"
- 课件工作台的"新增互动课程"按钮误导用户以为要创建新课程

## 2. 正确模型

### 2.1 用户视角

```
1. 教师进入 /teacher/courses → 看到所有课程列表
2. 点一门课 → 进入 CourseDetail,看到章节列表
3. 章节列表每一行显示:
   [排序] [标题] [类型标签: 📹视频/📊互动/📍线下/📝考试] [管理按钮]
4. 点管理按钮 → 进入该章节的子资源管理页面
5. 子资源管理页面顶部显示「课程 X / 章节 Y」
6. 在该页面创建/上传/配置该章节的子资源
```

### 2.2 数据流

```
章节表 (course_chapters)
  ├── chapterType = 'VIDEO'      → manage-videos   → POST /api/videos { chapterId }
  ├── chapterType = 'INTERACTIVE' → manage-slides   → POST /api/courses/{courseId}/slides/upload?chapterId={id}
  ├── chapterType = 'OFFLINE'     → manage-offline   → POST /api/chapters/{chapterId}/offline-sessions
  └── chapterType = 'EXERCISE'    → manage-exam      → POST /api/exercises { chapterId }
```

## 3. 改动范围

### 3.1 (可选|延迟) `<ChapterTypeSelector>` 公用组件

**暂不实现。** 本轮方案中 4 个新路由从 route.params 直接注入 courseId+chapterId,不需要此组件。组件留作将来(如独立菜单入口场景)使用,不在此 PR 实现。

具体实现时机:当从非 CourseDetail 入口(如全局菜单直接点"添加视频")需要选课程/章节时,再补建此组件。设计参考 `DynamicTableEditor.vue` 的单文件模式。

### 3.2 新增 4 个按章节管理的页面(直接复用现有页面)

**核心原则**: 不新建 4 个新的页面文件。新增路由直接指向**已有页面组件**,通过 route params 差异化。已有页面组件读取 `route.params` 中的 `courseId` + `chapterId` 自动进行范围限制。

| 新增路由 | 指向的已有页面 | 页面需要修改 |
|---------|---------------|-------------|
| `/teacher/courses/:courseId/chapters/:chapterId/manage-videos` | `VideoList.vue` | +5 行:读取 `route.params.chapterId`(已有 `courseIdFromRoute` 逻辑);传给 `searchForm.chapterId` 默认值 |
| `/teacher/courses/:courseId/chapters/:chapterId/manage-slides` | `SlideManage.vue` | +3 行:读取 `route.params.chapterId`;上传时透传给 `uploadSlide()` 函数 |
| `/teacher/courses/:courseId/chapters/:chapterId/manage-offline` | `TeacherOfflineSessions.vue` | +2 行:读取 `route.params.courseId`;面包屑中用 `getCourseById(courseId)` 显示课程标题 |
| `/teacher/courses/:courseId/chapters/:chapterId/manage-exam` | `ExamList.vue` | +5 行:读取 `route.params.chapterId`;智能组卷时传给 `ExamGenerateRequest.chapterId` |

**路由注册示例**:
```js
// router/index.js: 新增 4 个路由,复用现有页面组件
{ path: '/teacher/courses/:courseId/chapters/:chapterId/manage-videos',
  component: () => import('../views/courses/VideoList.vue'),
  meta: { requiresAuth: true, roles: ['TEACHER', 'ADMIN'] } },
// ... 其余 3 个类似
```

### 5.2 API 层: `uploadSlide` 函数加 chapterId 参数

**文件**: `micro-course-admin/src/plugins/interactive/api/slide.js`

```js
// 原: export function uploadSlide(courseId, file, onProgress)
// 新:
export function uploadSlide(courseId, file, onProgress, chapterId) {
  const fd = new FormData()
  fd.append('file', file)
  if (chapterId) fd.append('chapterId', chapterId)   // 可选
  return request({
    method: 'POST',
    url: `/courses/${courseId}/slides/upload`,
    data: fd,
    timeout: 300000,
    onUploadProgress: onProgress
  })
}
```
