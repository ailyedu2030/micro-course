## 4. 权限冲突与例外

### 4.1 教师同时也是学生的场景

**问题**：教师身份的用户也可能选课学习，成为某课程的学生。

**解决方案**：采用**身份叠加**而非身份切换。

```
用户身份组合：
- 纯 STUDENT：只能学习，不能建课
- 纯 TEACHER：只能建课，不能学习（除非也选了课）
- TEACHER + STUDENT：同时具有两种权限集合
  - 权限判定：取并集而非交集
  - 数据范围：在不同上下文中使用不同身份
```

**实现要点**：
1. 用户角色字段支持多角色（数据库存储为逗号分隔或单独关联表）
2. 权限判定时根据操作上下文自动识别身份：
   - 创建课程时 → 使用 TEACHER 身份
   - 选课学习时 → 使用 STUDENT 身份
   - 查看自己课程数据时 → 使用 TEACHER 身份
3. API 层需明确传递 `context=TEACHER|STUDENT`，或通过请求路径区分

**SpEL 表达式示例**：
```java
// 教师在创建课程场景下，验证教师身份
@PreAuthorize("hasRole('TEACHER') and " +
    "@permissionEvaluator.isCourseOwner(#courseId, #teacherId, authentication)")

// 教师在选课学习场景下，验证学生身份（需先确认该用户有STUDENT角色且已选课）
@PreAuthorize("hasRole('STUDENT') and @permissionEvaluator.isEnrolled(#courseId, authentication)")
```

---

### 4.2 管理员接管课程内容

**问题**：管理员是否可以编辑任意课程内容（如代教师修改视频、题目）？

**解决方案**：管理员**不可**直接编辑课程内容，遵循最小权限原则。

```
权限边界：
- ADMIN 可执行：审核、发布/下架、删除、查看全部数据
- ADMIN 不可执行：修改课程内容、编辑章节、上传视频、修改题目

例外情况（需额外授权）：
- 紧急情况（如课程内容违规需立即处理）：ADMIN 可执行 DELETE
- 课程转移（教师离职）：需教学院长审批后，由 ADMIN 执行课程负责人变更
```

**实现要点**：
1. 管理员角色不自动继承课程编辑权限
2. 课程内容编辑权限严格绑定 `course.teacher_id`
3. 系统配置、系统公告等基础设施操作属于 ADMIN 专属

**代码示例**：
```java
// 课程更新权限检查
@PreAuthorize("@permissionEvaluator.canUpdateCourse(#courseId, authentication)")

// PermissionEvaluator 实现
public boolean canUpdateCourse(Long courseId, Authentication auth) {
    User user = getCurrentUser(auth);
    if (user.getRole().contains("ADMIN")) return false; // 管理员无课程编辑权限
    Course course = courseRepository.findById(courseId);
    return course.getTeacherId().equals(user.getId());
}
```

### 4.3 教师跨课程查看

**问题**：教师 A 能否查看教师 B 创建的课程（仅查看，不编辑）？

**解决方案**：默认**不允许**，特殊情况例外。

```
默认规则：
- 教师只能查看自己创建的课程
- 教师不能查看其他教师的课程详情

例外情况：
1. 教务处查看：ACADEMIC 可查看全部课程统计
2. 管理员预览：ADMIN 可用"学生视角"预览任意课程
3. 课程分享：教师可主动将课程链接分享给同事（需显式操作）
4. 教学研讨：同一院系的教师可互相查看课程结构（需院系配置）
```

**实现要点**：
1. 课程详情 API 需验证 `course.teacher_id` 或角色权限
2. 不提供"查看他人课程"的公开 API
3. 日志记录教师查看非本人课程的审计事件

### 4.4 选课后自动获得讨论区权限

**问题**：学生选课后是否自动获得讨论区发帖权限？

**解决方案**：是的，选课后自动获得。

```
规则：
- 选课状态为 APPROVED 的学生，自动获得课程讨论区发帖权限
- 选课状态为 PENDING（待审批）的学生，仅可查看，不能发帖
- 退课后自动失去发帖权限（历史帖子保留）

教师权限：
- 课程创建者（TEACHER）自动获得讨论区管理权限（置顶、精华、删帖）
- 管理员（ADMIN）也有讨论区管理权限
```

**实现要点**：
```java
// 讨论发帖权限检查
@PreAuthorize("@permissionEvaluator.canPostDiscussion(#courseId, authentication)")

public boolean canPostDiscussion(Long courseId, Authentication auth) {
    User user = getCurrentUser(auth);
    // 教师身份：课程创建者
    if (user.getRole().contains("TEACHER")) {
        Course course = courseRepository.findById(courseId);
        return course.getTeacherId().equals(user.getId());
    }
    // 学生身份：已选课且状态为APPROVED
    if (user.getRole().contains("STUDENT")) {
        Enrollment enrollment = enrollmentRepository
            .findByUserIdAndCourseIdAndStatus(user.getId(), courseId, "APPROVED");
        return enrollment != null;
    }
    return false;
}
```

### 4.5 数据删除与审计

**问题**：用户删除后，其数据如何处理？

**解决方案**：软删除 + 保留审计。

```
规则：
1. 用户删除（status=0）后：
   - 用户信息保留（脱敏）用于审计
   - 该用户的发帖、评论保留（显示"已注销用户"）
   - 该用户的答题记录保留（用于统计分析）
   - 该用户的选课记录保留（用于成绩追溯）

2. 内容删除（课程/视频/题目）：
   - 管理员可删除，但保留操作日志
   - 删除后内容不可恢复（物理删除前需经审批流程）

3. 附件删除：
   - 上传者可删除
   - 管理员可删除任意附件
   - 删除前检查是否被其他内容引用
```

#### 5.2.2 PermissionEvaluator 实现

```java
@Component
public class CoursePermissionEvaluator implements PermissionEvaluator {

    @Autowired private CourseRepository courseRepository;
    @Autowired private EnrollmentRepository enrollmentRepository;
    @Autowired private UserRepository userRepository;

    @Override
    public boolean hasPermission(Authentication auth, Object targetDomainObject, Object permission) {
        if (auth == null || targetDomainObject == null) return false;
        String permissionStr = (String) permission;
        User user = getCurrentUser(auth);

        if (targetDomainObject instanceof Course) {
            Course course = (Course) targetDomainObject;
            return hasCoursePermission(user, course, permissionStr);
        }

        return false;
    }

    @Override
    public boolean hasPermission(Authentication auth, Serializable targetId, String targetType, Object permission) {
        if (auth == null || targetId == null || permission == null) return false;
        User user = getCurrentUser(auth);
        String permissionStr = (String) permission;

        if ("course".equals(targetType)) {
            Course course = courseRepository.findById((Long) targetId);
            return hasCoursePermission(user, course, permissionStr);
        }

        return false;
    }

    private boolean hasCoursePermission(User user, Course course, String permission) {
        // ADMIN：无课程内容编辑权限，但有管理权限
        if (user.isAdmin()) {
            return permission.startsWith("COURSE_MANAGE"); // 审核、发布等
        }

        // TEACHER：仅限创建者
        if (user.isTeacher() && course.getTeacherId().equals(user.getId())) {
            return permission.startsWith("COURSE_") ||
                   permission.startsWith("CHAPTER_") ||
                   permission.startsWith("VIDEO_") ||
                   permission.startsWith("QUESTION_") ||
                   permission.startsWith("EXERCISE_");
        }

        // STUDENT：已选课可读
        if (user.isStudent() && isEnrolled(user.getId(), course.getId())) {
            return permission.startsWith("COURSE_READ") ||
                   permission.startsWith("VIDEO_PLAY") ||
                   permission.startsWith("EXERCISE_TAKE");
        }

        return false;
    }

    public boolean isEnrolled(Long userId, Long courseId) {
        return enrollmentRepository
            .findByUserIdAndCourseIdAndStatus(userId, courseId, "APPROVED") != null;
    }

    public boolean isCourseOwner(Long courseId, Long teacherId) {
        Course course = courseRepository.findById(courseId);
        return course != null && course.getTeacherId().equals(teacherId);
    }
}
```
