# 微课平台 · 项目结构规范

> **本规范是项目结构宪法，AI 开发与人类开发均须遵守。**
> 违反规则的代码，不通过审查，退回重写。

---

## 1. 完整目录树

```
微课平台/ # 项目根目录
├── micro-course-api/                      # 后端：Spring Boot 3 + Java 17 + Maven + MyBatis-Plus
│   ├── pom.xml
│   └── src/main/java/com/microcourse/
│       ├── MicroCourseApplication.java # 启动类（必须放在根包下）
│       ├── config/                       # 全局配置（CORS、Security、Jackson、Swagger）
│       ├── controller/                    # REST Controller（只做参数校验和调用，不写业务逻辑）
│       ├── service/                       # 业务逻辑接口
│       │   └── impl/ # 业务逻辑实现
│       ├── repository/                   # MyBatis-Plus Mapper（interface + XML）
│       ├── entity/                       # @TableName 实体（JPA/MyBatis-Plus 注解）
│       ├── dto/                          # 请求 DTO / 响应 DTO（禁止 Entity 直接暴露）
│       ├── enums/                        # 枚举类（状态码、业务枚举）
│       ├── exception/                    # 全局异常处理 + 错误码定义
│       └── util/                         # 工具类（Date、String、JWT 等）
├── micro-course-admin/                    # 前端：admin 管理后台
│   ├── package.json
│   ├── vite.config.ts
│   └── src/
│       ├── api/                          # axios 接口封装（按模块分目录）
│       ├── views/                        # 页面组件（按模块分目录）
│       ├── components/                   # 可复用组件（全局组件放此目录）
│       ├── router/                       # 路由配置（index.ts + 模块级路由分文件）
│       ├── store/                        # Pinia 状态管理（按模块分目录）
│       └── utils/                        # 工具函数
├── docs/                                 # 文档目录（只放文档，❌ 禁止放代码）
├── scripts/                             # 部署/运维脚本（Shell/Ansible 等）
└── docker/                              # Dockerfile + docker-compose.yml
```

### 目录职责定义

| 目录 | 职责 | 允许放入的文件类型 |
|------|------|-------------------|
| `micro-course-api/src/main/java/…/controller/` | 接收 HTTP 请求，参数校验，调用 Service | `.java` |
| `micro-course-api/src/main/java/…/service/` | 所有业务逻辑 | `.java` |
| `micro-course-api/src/main/java/…/repository/` | 数据访问（MyBatis-Plus Mapper） | `.java` + `.xml` |
| `micro-course-api/src/main/java/…/entity/` | 数据库表映射实体 | `.java` |
| `micro-course-api/src/main/java/…/dto/` | 请求/响应数据传输对象 | `.java` |
| `micro-course-api/src/main/java/…/enums/` | 枚举常量 | `.java` |
| `micro-course-api/src/main/java/…/exception/` | 全局异常 + 错误码 | `.java` |
| `micro-course-api/src/main/java/…/util/` | 工具类 | `.java` |
| `micro-course-admin/src/` | 前端工程全部源码 | `.vue` / `.ts` / `.js` |
| `docs/` | 项目文档 | `.md` / `.pdf` / `.docx` |
| `scripts/` | 运维脚本 | `.sh` / `.ps1` / `.yml` |
| `docker/` | 容器化配置 | `Dockerfile` / `docker-compose.yml` |

## 2. 负面清单（AI 执行时逐条检查）

> **所有规则均为「禁止」格式，AI 在创建文件前须逐条核验。**

###2.1 文件位置规则

```
❌ 禁止：在项目根目录（微课平台/）下直接创建 .java / .vue / .ts / .js 文件
❌ 禁止：将后端 Java 文件放在 docs/、scripts/、docker/、micro-course-admin/ 目录下
❌ 禁止：将前端 Vue/TS 文件放在 micro-course-api/、docs/、scripts/、docker/ 目录下
❌ 禁止：在 micro-course-api/src/main/java/com/microcourse/ 之外创建 Java 源文件
❌ 禁止：在 micro-course-admin/src/views/ 之外创建页面级 .vue 组件
❌ 禁止：在 micro-course-admin/src/components/ 之外创建全局可复用组件
❌ 禁止：在 micro-course-api/src/main/java/com/microcourse/entity/ 之外创建 @Entity 实体类
❌ 禁止：在 micro-course-api/src/main/java/com/microcourse/之外创建 Controller
```

### 2.2 重复定义规则

```
❌ 禁止：创建已存在的 Entity（先执行 grep 确认不存在）
❌ 禁止：创建已存在的 Controller（先执行 grep 确认不存在）
❌ 禁止：创建已存在的 Service 接口（先执行 grep 确认不存在）
❌ 禁止：创建已存在的 Mapper（先执行 grep 确认不存在）
❌ 禁止：创建已存在的 DTO（先执行 grep 确认不存在）
❌ 禁止：创建已存在的 Vue 页面组件（先执行 grep 确认不存在）
❌ 禁止：手动创建数据库 migration 而不更新数据字典 docs/数据字典.md
```

### 2.3 分层职责规则

```
❌ 禁止：Controller 中写业务逻辑（逻辑必须放入 Service 层）
❌ 禁止：Controller 直接返回 Entity（必须通过 DTO 转换后返回）
❌ 禁止：Service 中直接暴露 Entity（通过 DTO 转换后再暴露）
❌ 禁止：Repository 层写业务逻辑（只做数据访问）
❌ 禁止：前端 API 封装中硬编码 baseURL（必须使用环境变量 VITE_API_BASE_URL）
❌ 禁止：手动拼接 SQL 字符串（必须使用 MyBatis-Plus 条件构造器 QueryWrapper/LambdaQueryWrapper）
❌ 禁止：在后端直接返回数据库实体给前端（必须经 DTO 转换）
```

### 2.4 代码风格规则

```
❌ 禁止：Java 类名使用 camelCase（正确：UserController，错误：userController）
❌ 禁止：Java 字段名使用 snake_case（正确：userId，错误：user_id）
❌ 禁止：数据库字段名使用 camelCase（正确：user_id，错误：userId）
❌ 禁止：Vue 组件名使用 kebab-case 文件名（组件文件应为 PascalCase：UserList.vue）
❌ 禁止：Vue 路由路径使用 PascalCase（正确：/user-list，错误：/userList）
❌ 禁止：前后端字段命名不一致（DB 用 snake_case，Java 用 camelCase，DTO 保持与前端一致用camelCase）
❌ 禁止：Service 使用 @Transactional 但不处理 rollbackFor
❌ 禁止：Controller 使用 @ResponseBody 返回 POJO 时不配置消息转换器
```

### 2.5 架构边界规则

```
❌ 禁止：后端绕过 Service 直接调用 Repository（所有数据访问经 Service）
❌ 禁止：前端组件内直接调用 axios（必须经 api/ 层封装）
❌ 禁止：在 Vue 组件中直接操作 Pinia store状态（通过 actions 方法）
❌ 禁止：在微课平台项目内引入未在 pom.xml / package.json 声明的依赖
```

## 3. 预检流程

> **每次创建新文件前，必须执行以下检查。通过后才能创建文件。**

### 3.1 Entity 预检

```bash
# 检查同名 Entity 是否已存在
grep -r "class XxxEntity" micro-course-api/src/main/java/com/microcourse/entity/ || echo "Entity 不存在，可创建"
```

### 3.2 Controller 预检

```bash
# 检查同名 Controller 是否已存在
grep -r "class XxxController" micro-course-api/src/main/java/com/microcourse/controller/ || echo "Controller 不存在，可创建"
```

### 3.3 Service 预检

```bash
# 检查同名 Service 是否已存在
grep -r "class XxxService" micro-course-api/src/main/java/com/microcourse/service/ || echo "Service 不存在，可创建"
```

### 3.4 Mapper 预检

```bash
# 检查同名 Mapper 是否已存在
grep -r "interface XxxMapper" micro-course-api/src/main/java/com/microcourse/repository/ || echo "Mapper 不存在，可创建"
```

### 3.5 前端组件预检

```bash
# 检查同名 Vue 组件是否已存在
grep -r "XxxList.vue\|XxxForm.vue\|XxxDialog.vue" micro-course-admin/src/views/ || echo "组件不存在，可创建"
```

### 3.6 API 预检

```bash
# 检查同名 API 方法是否已存在
grep -r "export.*getXxx\|export.*postXxx" micro-course-admin/src/api/ || echo "API 不存在，可创建"
```

### 3.7 目录合规性检查

```bash
# 后端文件必须在 micro-course-api/ 下
if echo "$file_path" | grep -qv "^micro-course-api/"; then
    echo "❌ 错误：后端文件必须放在 micro-course-api/ 目录下"
    exit 1
fi

# 前端文件必须在 micro-course-admin/ 下
if echo "$file_path" | grep -qv "^micro-course-admin/"; then
    echo "❌ 错误：前端文件必须放在 micro-course-admin/ 目录下"
    exit 1
fi
```

## 4. 命名规范

### 4.1 Java 命名

| 类型 | 规范 | 示例 |
|------|------|------|
| 类名 | PascalCase | `UserController`、`CourseServiceImpl` |
| 方法名 | camelCase | `getUserById`、`saveCourse` |
| 字段名 | camelCase | `userId`、`courseName` |
| 常量 | UPPER_SNAKE_CASE | `DEFAULT_PAGE_SIZE`、`MAX_UPLOAD_SIZE` |
| 包名 | 全小写 dot 分隔 | `com.microcourse.controller` |
| 枚举值 | UPPER_SNAKE_CASE | `COURSE_STATUS_PUBLISHED` |

### 4.2 数据库命名

| 类型 | 规范 | 示例 |
|------|------|------|
| 表名 | 全小写 + snake_case | `users`、`course_chapter`（保留） |
| 字段名 | 全小写 + snake_case | `user_id`、`created_at` |
| 主键 | `id` 或 `{table}_id` | `id`、`user_id` |
| 外键 | `{referenced_table}_id` | `course_id`、`user_id` |
| 索引 | `idx_{table}_{column}` | `idx_users_phone` |
| 唯一索引 | `uk_{table}_{column}` | `uk_users_phone` |

### 4.3 前端命名

| 类型 | 规范 | 示例 |
|------|------|------|
| Vue 组件文件 | PascalCase.vue | `UserList.vue`、`CourseDialog.vue` |
| Vue 组件名 | PascalCase | `UserList`、`CourseDialog` |
| 路由路径 | kebab-case | `/user-list`、`/course-detail` |
| API 方法 | 小写 +动词 | `getUserList`、`postCourseCreate` |
| 目录名 | kebab-case | `user-management`、`course-info` |
| CSS 类名 | kebab-case | `user-list-container` |
| 常量/枚举 | UPPER_SNAKE_CASE | `API_BASE_URL`、`PAGE_SIZE` |

### 4.4 API 路径规范

```
# 后端 REST API 路径
GET    /api/users # 获取用户列表
GET    /api/users/{id}         # 获取单个用户
POST   /api/users              # 创建用户
PUT    /api/users/{id}         # 更新用户
DELETE /api/users/{id}         # 删除用户

# 前端 API 封装（对应上述路径）
getUserList(params)
getUserById(id)
postUserCreate(data)
putUserUpdate(id, data)
deleteUserById(id)
```

## 5. 分层架构约束

```
┌─────────────────────────────────────────────────────┐
│                    Vue Components │
│         (micro-course-admin/src/views/)             │
└──────────────────────┬──────────────────────────────┘
                       │ 调用
┌──────────────────────▼──────────────────────────────┐
│                   API Layer │
│         (micro-course-admin/src/api/)               │
└──────────────────────┬──────────────────────────────┘
                       │ HTTP
┌──────────────────────▼──────────────────────────────┐
│               Controller Layer │
│    (micro-course-api/.../controller/)              │
│        ❌ 禁止写业务逻辑 │
└──────────────────────┬──────────────────────────────┘
                       │ 调用
┌──────────────────────▼──────────────────────────────┐
│                Service Layer                        │
│     (micro-course-api/.../service/)                 │
│         ✅ 所有业务逻辑在此 │
└──────────────────────┬──────────────────────────────┘
                       │ 调用
┌──────────────────────▼──────────────────────────────┐
│              Repository Layer                       │
│      (micro-course-api/.../repository/)            │
│    ❌ 禁止写业务逻辑，只做数据访问                  │
└──────────────────────┬──────────────────────────────┘
                       │
┌──────────────────────▼──────────────────────────────┐
│                   PostgreSQL                         │
└─────────────────────────────────────────────────────┘
```

### DTO 转换规则

```
Entity（数据库） ──→ DTO（后端内部） ──→ JSON（HTTP响应） ──→ 前端接口类型
```

**每层只允许临近层访问，禁止跨层调用。**

## 6. 惩罚规则

| 违规类型 | 惩罚措施 |
|---------|---------|
| 文件放错目录 | 强制移动文件，退回重写，记违规一次 |
| 重复定义（已存在 Entity/Controller 等） | 拒绝创建，AI 须先做预检 |
| Controller写业务逻辑 | 代码审查不通过，退回 Service 层 |
| 硬编码 baseURL | 强制改为环境变量，拒绝直接提交 |
| 手动拼 SQL | 强制改为 MyBatis-Plus 条件构造器 |
| 前后端字段名不一致 | 要求对齐数据字典，统一命名 |
| 未做预检就创建文件 | 警告一次，连续两次自动触发规范重新学习 |

## 7. 附：数据字典维护约定

所有数据库表结构变更必须同步更新到 `docs/数据字典.md`，
格式示例：

```markdown
## users（用户表）

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | bigint | PK, AUTO_INCREMENT | 主键 |
| username | varchar(50) | NOT NULL, UNIQUE | 用户名 |
| phone | varchar(20) | UNIQUE | 手机号 |
| created_at | timestamp | DEFAULT CURRENT_TIMESTAMP | 创建时间 |
```

**每次 migration 必须同步更新数据字典，违者按惩罚规则处理。**
