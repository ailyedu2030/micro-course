## 审查元规则

每条 todo 必须产出 4 项产物：
1. **PASS/FAIL 判定**（不能"看过了"）
2. **代码证据**（具体文件:行号）
3. **P0-P3 分级**（依据用户体验优先原则）
4. **修复建议**（FAIL 项必须给出最小可行修复方案）
