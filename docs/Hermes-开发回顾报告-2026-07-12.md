## 三、根因分析

### 根因 1：前后端独立实现，未做集成测试

```
问题: uploadHtml() 路由到 /upload（FormData），但 /upload 做 ZIP 魔数校验拒 HTML
修复: 统一 /upload 端点按扩展名分支（.pptx → POI, .html → sanitize）
防止再发: 前后端 API 契约必须同步更新，CI 加契约检查
```

### 根因 2：拿 Jsoup Safelist.relaxed() 默认值未做安全剪枝

```
问题: data: URI、style 属性、target、id/class 全部默认允许
修复: 自定义 Safelist，禁止 4 类危险属性
防止再发: safelist 变更必须有安全审查（文档已加）
```

### 根因 3：iframe sandbox 语义误解

```
问题: 以为 allow-same-origin 是"沙箱"配置，实际是"允许同源脚本"（P0 XSS）
修复: sandbox="" 完全沙箱（禁用脚本/表单/同源/导航/弹窗）
防止再发: iframe sandbox 默认值必须显式注释说明
```

### 根因 4：NOT NULL 字段在新增写入路径时被遗忘

```
问题: uploadHtmlFile() 复制 upload(PPTX) 代码时，
     忽略了 setFileUrl() 和 setImageUrl()
修复: 加 setFileUrl('html:inline') 和 setImageUrl('html:no-image')
防止再发: INSERT 前 SELECT information_schema.columns WHERE is_nullable='NO'
```

### 根因 5：回滚脚本放在 Flyway 自动迁移目录

```
问题: V178 在 db/migration/ 中，Flyway 自动执行 → 回滚了 V177
修复: 移到 db/rollback/（手动执行）
防止再发: 回滚脚本永远不得在标准 migration 目录
```
