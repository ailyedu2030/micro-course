# 04 · 无障碍适配基线 + 老年用户/残障场景验收清单

> **生效日期**：2026-07-21
> **基准**：WCAG 2.1 Level AA（强制） + **Level AAA 关键 7 条**（推荐）
> **横向贯穿**：本基线覆盖 01 量化标准的**维度 D（视觉舒适度）**，并叠加老年/残障维度。

---

## 1. 总原则

> **微课平台不是"互联网 IT 应用"，是"教育基础设施"。
> 一位 70 岁的奶奶、外地务工的视障青年、读写障碍的中学生，都是我们的用户。**

| 用户群 | 能力假设 | 体验要求 |
|--------|---------|---------|
| **K12 学生** | 注意力短、易分心、对快捷键熟悉 | 一致性、即时反馈、游戏化、家长模式 |
| **职教 / 语言学员** | 时间碎片、地铁通勤、移动场景 | 离线容忍、低流量、声音清晰 |
| **老年大学** | 视力下降、动作慢、对 IT 陌生、不识字比例非零 | 大字号、高对比、语音提示、可电话求助 |
| **残障学员** | 视障 / 听障 / 运动障碍 / 读写障碍 | 屏幕阅读器、字幕、键盘完全可达 |

> **任何"为年轻人做的优化"不能以损害老年用户为代价，反之亦然。**

---

## 2. WCAG 2.1 AA 强制项（必过）

> 微课平台所有前台页面**必须**通过 axe-core 扫描零 violations。
> 强制规则如下（按 POET 四原则分组）：

### 2.1 可感知 Perceivable
| # | 规则 | 测试方法 |
|---|------|---------|
| P-1 | 所有非装饰图像必须有 `alt` | axe image-alt |
| P-2 | 视频必须有字幕或文字稿（重要课件） | 人工抽检 + 字幕供应商验收 |
| P-3 | 颜色不是传达信息的唯一方式 | 色彩对比 + 形态学独立编码 |
| P-4 | 文本对比度 ≥ 4.5:1（≥ 18pt 时 ≥ 3:1） | axe color-contrast |

### 2.2 可操作 Operable
| # | 规则 | 测试方法 |
|---|------|---------|
| O-1 | 所有交互**键盘可达**（Tab 顺序合理） | 键盘走查 |
| O-2 | 焦点可见（`:focus-visible` 明确样式） | 视觉 + axe |
| O-3 | 移动/缩放/横向滚动无锁定 | Chrome DevTools |
| O-4 | 点击目标 ≥ 44×44 CSS 像素 | axe target-size |
| O-5 | 跳过链接存在（"跳到主内容"） | 人工 + axe |

### 2.3 可理解 Understandable
| # | 规则 | 测试方法 |
|---|------|---------|
| U-1 | 页面语言正确声明 `<html lang="zh-CN">` | axe html-has-lang |
| U-2 | 表单错误描述明确 + 提供修复建议 | 表单走查 |
| U-3 | 表单有可见标签（不是仅 placeholder） | axe label |

### 2.4 健壮 Robust
| # | 规则 | 测试方法 |
|---|------|---------|
| R-1 | HTML 解析一致（无重复 id） | axe |
| R-2 | ARIA 使用正确（无冲突） | axe + NVDA |

## 3. 老年用户专项基线

> 60+ 用户占比 35% 的客群，我们做四项专项：

### 3.1 字号与可读性
- 正文 ≥ **18px**（推荐 AAA：≥ 19px）
- 行高 ≥ **1.7**
- 最大宽度 ≤ 70 中文字符 / 行
- 中文字体支持：默认 + 思源宋体回退（避免 Linux 渲染瘦小）

### 3.2 操作容错
- 点击区域 ≥ **56×56 px**
- 危险操作必须**二次确认**（带图标+文字）
- 文字输入框**始终**带 label
- "返回" 按钮始终可见

### 3.3 流程简化
- 主任务路径 ≤ **3 步**（KPI 维度 C）
- 不可逆操作（删除）必须 explicit 二次确认 + 提供"撤销"提示
- 客服电话按钮在所有页面**底部固定可见**

### 3.4 多通道求助
- 应用内 "联系客服" 全局可达
- 客服电话单独展示，字号 ≥ **24px**
- 微信客服号有图标 + 大字号

## 4. 残障场景专项

### 4.1 视障（屏幕阅读器优先）
- 全部交互元素必须有 ARIA label
- 表单输入必须关联 label（for + id）
- 模态框焦点自动落入，且 `Esc` 可关闭
- 动态内容（SPA 路由切换）必须 announce
- 测试工具：NVDA（Windows） + VoiceOver（macOS / iOS）

### 4.2 听障
- 视频课件**必须带字幕**
- 音频课件**必须带文字稿**
- 任何声音提示必须有视觉替代（图标）

### 4.3 运动障碍
- 全部功能键盘可达
- 不依赖悬停（hover）触发关键操作
- 长按 / 双击 / 拖拽 必须提供替代方式（键盘）

### 4.4 读写障碍（dyslexia）
- 避免左右对齐（justified）段落
- 字符间距 ≥ 0.12em
- 提供 dyslexia 友好字体切换（开思源宋体）
- 段落 ≤ 4 行

### 4.5 认知 / 学习障碍
- 语言简单化（首屏关键信息 ≤ 14 岁阅读水平）
- 一致性（同一动作在所有页面路径相同）
- 进度可见

## 5. 实施技术清单

### 5.1 前端
- 复用 `element-plus` 的 a11y 支持，使用 `<el-button>` 而非 `<div>`
- 富文本编辑器：工具栏有 aria-label
- 路由切换：监听并广播给屏幕阅读器
- 视频播放器：原生 `<video controls>` + `<track kind="captions">`

### 5.2 设计 Token
```css
:root {
  --font-base: 16px;
  --font-base-elderly: 18px;
  --line-height-base: 1.6;
  --line-height-elderly: 1.7;
  --color-text: #1f2329;     /* 对比度 ≥ 13:1 */
  --color-bg: #ffffff;       /* 对比度 ≥ 13:1 */
  --color-link: #1554ff;
  --focus-ring: 3px solid #facc15;
}
```

### 5.3 测试
- axe-core 在 PR 中必跑，FAIL = 不合并
- Playwright a11y job（已有 a11y.config.js）
- 每月人工走查 ≥ 1 个主流程

## 6. 上线前 a11y 验收清单

> **任何 PR 修改涉及前台代码，必须勾选以下清单。**

```
PR 标题: feat/fix/...
前置检查（CI 自动）：
  [x] axe-core 0 violations
  [x] Playwright a11y job 通过
  [x] 触摸目标 ≥ 44px

人工走查（作者必做）：
  [x] 键盘 Tab 走查：顺序合理，不卡死
  [x] 焦点可见：Tab 每个目标都有明确 focus ring
  [x] 文本对比度：≥ 4.5:1（暗色主题另行验证）
  [x] 字号 ≥ 16px（正文）/ ≥ 18px（老年模式）
  [x] 表单有 label，无 placeholder 替代
  [x] 模态/抽屉 Esc 关闭 + 焦点回收

复杂场景（修改富文本/上传/拖拽）：
  [x] 屏幕阅读器走读（NVDA or VoiceOver）
  [x] 拖拽有 keyboard 替代
  [x] 老年字号模式样式不破

挂起项（可申请例外，必须记录）：
  □ 旧 slide_pages 相关，已标记 wontfix + 替代方案 <链接>
```

完整版清单：[templates/a11y-acceptance-checklist.md](./templates/a11y-acceptance-checklist.md)

## 7. 回归节奏

| 频率 | 动作 |
|------|------|
| **每个 PR** | axe-core + Playwright a11y job |
| **每周一复盘前** | 全站 axe 扫描生成报告 |
| **每月末** | 真实键盘 + 屏幕阅读器走查 1 个主流程 |
| **每季度** | 外呼 ≥ 3 位老年用户实际操作 + NVDA 走查 |
| **每年** | 第三方 a11y 审计（预算允许时） |

---

## 8. 失效红线

- 任何新页面上线 axe-core 检查未通过
- 主任务（登录 / 上传课件 / 播放视频）键盘无法完成
- 老年模式开启后正文仍 < 18px
- 视频课件发布时无字幕
