# 06 · 项目用户体验健康度报告模板

> **生效日期**：2026-07-21
> **频率**：月度固定 + 季度深度（季度版本在月度基础上扩展）
> **目标读者**：项目相关方（业务方 / 管理层 / 团队）
> **签发人**：项目总负责人（UX 负责人）
> **范围**：覆盖 [01 量化标准](./01-metrics-standard.md) 全部 5 维度 + [03 反馈 SLA](./03-feedback-sla.md) 全部 KPI

---

## 1. 报告章节（固定 8 节）

```
1. 摘要
2. 5 维度指标健康度（一页红黄绿）
3. 反馈 SLA 月度表现
4. 案例精选（≤3）
5. 应急 / 事故
6. 老年 & 残障专项
7. 下月重点
8. 数据附录（图表 + 原始指标）
```

---

## 2. 月度报告骨架（v1）

```markdown
# 微课平台 · 用户体验健康度月报 #Y#M
报告期: 2026-0X-0X ~ 2026-0Y-0Y
签发: 项目总负责人
状态: 草稿 / 已发布

## 1. 摘要（≤200 字）
- 整体健康度评级: 🟢 / 🟡 / 🔴
- 本月最大改善: <指标 / 案例>
- 本月最大风险: <指标 / 案例>
- 月活独立体验影响: <用户数 / 主任务次数>
- 已闭环: <反馈数 / 全部>
- 满意度: <NPS 或 自定义指标>

## 2. 5 维度指标
（以一张红黄绿密度图 + 5 行表格呈现，见 01）

| 维度 | 关键指标 | 阈值 | 实测 | 同比 | 趋势 | 评价 |
|------|---------|------|------|------|------|------|
| A 加载 | FCP / LCP / TTI | 1.5 / 2.5 / 3.5 | <...> | | | 🟢 |
| B 交互 | INP / API p95 | 200 / 800 | | | | |
| C 易用 | 任务完成率 | 95% | | | | |
| D 视觉 | WCAG AA 通过率 | 100% | | | | |
| E 恢复 | 平均恢复 | ≤5min | | | | |

## 3. 反馈 SLA 月度
- 总反馈: X 条
- 首响应达成率: X%
- 定级达成率: Y%
- 方案达成率: Z%
- 回访达成率: W%
- 同比增长/下降: 列出 top 3 变化指标
- 闭环案例数 / 关闭含回访 / WontFix（必须给原因）

## 4. 案例精选 (3 个)
按影响最大、最近整改完成、未来趋势最值得关注，三个角度各一。

### 案例 4.1
摘要 + 数据 + 整改 + 用户反馈

## 5. 应急/事故
- 本月 UX-P0: X 次
- UX-P1: Y 次
- 已发 Postmortem: <列表>
- 防再发闭环率: Z%

## 6. 老年 & 残障专项
- 老年外呼: N 人 / 闭环率
- a11y axe-core 通过率（生产环境）
- 屏幕阅读器走查覆盖: <N 个主流程>
- 改善前后对比截图

## 7. 下月重点
- 改进项 1: …（已开 issue）
- 改进项 2: …（已开 issue）
- 风险预案: …（白名单 + 回滚开关）

## 8. 数据附录
- 全部 22 个指标的周趋势表（4 周 × 22 指标）
- 反馈池汇总表
- 关键事件时间线
- 度量原始数据 CSV 链接
```

---

## 3. 季度深度报告（每月 + 三段扩展）

```
A. 5 维度趋势（3 个月 × 22 指标）：同环比、热力图
B. 重大版本/发布 UX 影响：每个 GA 版本一段
C. 用户画像变化：老年用户占比、移动端占比、跨设备完成率
D. 长期战略对照：与 5 维度阈值的全年目标距离
```

---

## 4. 数据采集口径（确保月度间可比）

| 项 | 定义 |
|----|------|
| 报告期 | 自然月（每月 1 日 00:00 ~ 月末 23:59 UTC+8） |
| FCP/LCP/INP | 全站用户访问加权均值，使用 75 分位数；剔除 CI 与开发环境 |
| API p95/p99 | Grafana 中与微课业务请求精确匹配的 p50/p95/p99 |
| 任务完成率 | 实验室 8 任务每月抽样实测；不计入线上 |
| 满意度 | 季度一次 ≥ 30 用户结构化访谈 + NPS 公式 |
| 老年外呼 | ≥60 岁真实用户，使用真实设备，完成主任务 |

---

## 5. 发布与归档

- 月报 commit 到：`docs/governance/ux-governance/health-reports/YYYY-MM.md`
- 季度报告 commit 到：`docs/governance/ux-governance/health-reports/YYYY-Q?.md`
- 归档一周后追加到 `docs/CHANGELOG.md` UX 部分
- 不公开给人**之外**的对象（内部使用），但**核心结论**可发到值班群

## 7. 失效红线

- 月报延迟 ≥ 7 天
- 缺失任意 5 维度数据
- 没有下月重点
- 没有闭环反馈案例
- 没签发人签字
