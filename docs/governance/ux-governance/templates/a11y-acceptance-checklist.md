# 上线前 · 无障碍 (a11y) 验收清单

> **必填**：任何修改前台代码的 PR
> **位置**：PR description 中（也可作为 issue checklist）
> **未勾选** = 不予合并（除非填写"申请例外" + 责任人承认）

```markdown
## a11y 验收清单 (PR #xxxx · 标题)

### 1. CI 自动检查 (必填)
- [ ] axe-core 0 violations (PR bot 报告)
- [ ] Playwright a11y job 通过 (CI green)
- [ ] 触摸目标 ≥ 44×44px (Lighthouse / axe target-size)
- [ ] 富文本编辑器 toolbar 不破坏 ARIA

### 2. 人工键盘走查 (作者必做)
- [ ] Tab 顺序：从侧栏 → 内容 → 操作 → 提示
- [ ] 焦点环可见 (`:focus-visible`)
- [ ] 不存在焦点陷阱（焦点能跳出去）
- [ ] Esc 关闭弹窗/抽屉/对话框
- [ ] Skip Link ("跳到主内容") 生效

### 3. 视觉对比度 (axe color-contrast)
- [ ] 正文 ≥ 4.5:1 (默认主题)
- [ ] 正文 ≥ 4.5:1 (暗色主题)
- [ ] 老年模式下 ≥ 7:1 (AAA)
- [ ] 错误态颜色 + 图标，不依赖单独颜色

### 4. 表单可达性
- [ ] 每个 input 有 `<label for>` 或 `aria-label`
- [ ] 错误信息紧贴字段，并 `role="alert"`
- [ ] 占位符 (placeholder) 不作为唯一标签
- [ ] 提交后焦点回到首个错误字段

### 5. 富文本 / 上传 / 拖拽
- [ ] Quill 工具栏 aria-label 完整
- [ ] 文件上传可键盘触发
- [ ] 拖拽有键盘替代（方向键 + 空格确认）
- [ ] 模态打开焦点自动落入

### 6. 多设备 / 多浏览器
- [ ] Chrome 最新
- [ ] Safari 最新
- [ ] Firefox 最新
- [ ] iOS Safari (移动)

### 7. 老年 / 残障场景
- [ ] 字号切换到"老年模式"(≥18px) 样式不破
- [ ] NVDA / VoiceOver 主流程走读（修改涉及交互必做）
- [ ] 客服电话按钮在所有新页面可见

### 8. 屏读体验 (新增/修改主流程必做)
- [ ] 路由切换 announce 给屏幕阅读器
- [ ] 错误提示通过 aria-live="polite" 通告

### 9. 申请例外（仅在不破坏红线前提下允许）
- 不适用 / 例外：

### 10. 责任人签字
- [ ] 作者签字: @<name>
- [ ] a11y owner 复核: @<name>
```

---

## 例外申请流程

1. 在 PR 中明示"申请例外"+ 填写"为何无法达标"
2. 上文 §1-3 任意红区不可豁免（WCAG AA 强制项）
3. 必须在 issue 中创建"防再发跟踪"，明确 N 个月内改进到 green
4. 项目总负责人签字 → 放行

---

## 失效红线
- PR 中该清单**未勾选**[1]CI 部分 → 不合并
- 任何新页面 axe 0 violations 未达到 → 不合并
- 老年/视障专项主任务失败 → 回滚
