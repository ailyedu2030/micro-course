# UX 治理值班排班表

> **首版生效**：2026-07-21（第 N 周期）
> **更新节奏**：每 4 周（一个月）轮换一次
> **缺席制度**：连续 2 周缺岗 = 视为放弃当周 SLA，下周自动换人
> **维护人**：项目总负责人 + 产品值班

## 🔒 UX 独立资源保障条款 (EXECUTIVE-PRINCIPLE § 三 资源保障)

> 本条款由 L0 铁律派生, **资源缺口直接视为 P1 治理事件**, 项目总负责人必须主动补齐.

### R1. 设计资源

- **首期**: ≥1 名全职 UX 设计师 (首期由总负责人兼任, **8-04 前**完成招聘)
- **备用**: ≥1 名外部 UX 顾问 (合同形式), 24h 响应紧急 UX 评审
- **运营预算**: 设计与开发**同等级并发节奏** (开发阻塞设计 = 重大治理事故)

### R2. 用户调研池 (依 [07-research-lab-sop.md](./07-research-lab-sop.md))

| 子人群 | 最低样本 | 季度回访 |
|--------|---------|---------|
| 老年大学 (≥60 岁) | 10 人 | ≥1 次 |
| 中职教师 | 10 人 | ≥1 次 |
| 6-12 年级学生 | 10 人 | ≥1 次 |
| 视障 / 运动障碍 | 5 人 | ≥1 次 |
| 月度回访 (一般用户) | 30 人 | ≥1 次 |

### R3. 用户行为分析 (依 [01-metrics-standard.md](./01-metrics-standard.md) §1)

| RUM 来源 | 数据点 |
|---------|--------|
| 前端 | FCP / LCP / CLS / INP / FID |
| 后端 | API p95 / p99 (Grafana / monitor-lint) |
| 上报 | 应用内反馈按钮 / 客服电话 / 微信公众号 / 教室现场 |

### R4. 应急预算

- 24h 可达的 UX 工单应急支持 1 名 (可与全职 UX 兼任)
- 外部 UX 顾问 24h 响应 (合同 already)

### R5. 缺口上报

- 任何资源不足由值班主动**月度报告**声明 + 立即向总负责人通报
- 总负责人必须**优先资源倾斜**填补 (依铁律只令 4)

### R5+. 资源倾斜强制触发器 (UX 攻坚 1 号动员令 § 6)

> 任何"可单方解决"的资源倾斜有专门触发器, 不必反复向总负责人说明.

| 触发情况 | 可单方批准 | 时效 |
|---------|----------|------|
| 招聘/外聘启动 | 总负责人签字 → HR 即起 | 24h |
| 跨部门借调 UX / 可用性资源 | 总负责人批准 → 目标部门配合 | 24h |
| 外包 UX 服务紧急采购 (≤ 5 万) | 总负责人签字即生效, 后补财务流程 | 48h |
| 工具采购 (Figma / NVDA 许可) | UX 值班申请 → 总负责人批 | 48h |
| 加急合规调研 (录制/录像) | 用户研究专员 → 总负责人秒批 | 4h |
| 月度回访样本扩充 (>30 → >100) | 用户研究专员申请 → 24h 批 | 24h |
| **拒绝的资源申请必填理由** | 总负责人 → 申请方反提清单 | 72h |

> **拒绝原则**: 任何拒绝必须附**书面理由 + 复审日期** (不批准的拒绝不算). 这就是"不予否决 = 视为批准"的反向默示规则.

### R5++. 跨部门 UX 仲裁通道 (UX 攻坚 1 号动员令 § 5)

**直接抄送总负责人**:
- 任何部门阻碍 UX 落地 → 24h 内回复
- UX 红线冲突 (合规/预算/排期/工期) → 4h 内裁决
- 跨部门资源对调 (急辞 / 招用 / 工具) → 24h 内批准
- 预算突破 ≤ 1 万时, 总负责人前置批准 → 财务后置
- 1 万 < X ≤ 5 万时, 总负责人批准 → 治理委员会 24h 备案
- X > 5 万时, 治理委员会重大事项审批 → 总负责人不可单方

**越级匿名通道**:
> 任何人发现"为赶工违反 UX"或"部门蓄意阻碍 UX 落地", 可**直接邮件报告总负责人**, 不经任何中间层. 报告通道:
> - 主: `ailyedu@outlook.com` (总负责人私有, 不开放订阅)
> - 备用: 公告 #ux-constitution 频道 (Slack/Slack-Like)
> - 4h 内必须确认收到, 24h 内给出处置方案
