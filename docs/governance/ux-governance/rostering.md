# UX 治理值班排班表

> **首版生效**：2026-07-21（第 N 周期）
> **更新节奏**：每 4 周（一个月）轮换一次
> **缺席制度**：连续 2 周缺岗 = 视为放弃当周 SLA，下周自动换人
> **维护人**：项目总负责人 + 产品值班

---

## 排班结构（一个周期 4 周）

| 角色 | 责任范围 | 必备人数 |
|------|---------|---------|
| 后端值班 | API 性能、缓存、限流、错误日志 | 1 |
| 前端值班 | FCP/LCP/INP、a11y bug、交互流畅 | 1 |
| 测试值班 | e2e、可用性测试、bug 跟踪、Recorder | 1 |
| 运维值班 | 监控告警、灰度开关、回滚执行 | 1 |
| 产品值班 | 用户故事、反馈通道值守、投诉闭环 | 1 |
| 客服/外呼轮值 | 接待老年用户、收集反馈、电话回访 | 1-2 |

> 总负责人（项目总负责人）兼任召集人 / IC。

## 每周复盘会时间

- **固定时间**：周一 16:00-17:00
- **远程链接**：待补
- **会议室**：待补
- **首次复盘**：启动后第二个周一（覆盖启动周的反馈）

---

## 排班 SOP

1. **每周一 09:00 前** 确认本周值班无变化
2. **如有人请假**：当日上午找到替补，未找到 = 召集人指派
3. **每月最后一周四** 排下个月排班表
4. **排班冲突**：召集人（项目总负责人）一票裁定
