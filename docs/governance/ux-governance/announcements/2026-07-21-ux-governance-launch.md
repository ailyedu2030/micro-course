## 一句话公告

---

## 启动背景

微课平台面对 K12 / 职教 / 语言 / 老年大学四类客群，**用户视角的"健康度"应当成为项目治理的第 7 套机制**，与既有的：

1. AGENTS.md (P0 生产安全铁律)
2. docs/开发流程-完整版.md
3. docs/发布管理.md
4. docs/PRODUCTION_SAFETY.md
5. monitoring/ (Prometheus + AlertManager + Grafana)
6. .github/workflows/auto-approve.yml (CI 门禁)

并列。本公告正式确立第 7 套机制：

> **用户体验治理体系（UX Governance）** — 量化标准 / 周复盘 / 反馈 SLA / 无障碍 / 应急 / 健康度报告 / 研究 SOP

## 与既有治理的边界

```
研发轴 (CODE - 已经成熟)
├─ 开发流程、PR/R5 review、auto-approve
├─ CI: 5 维度监控 + 监控 lint
└─ 发布管理、PRODUCTION_SAFETY

体验轴 (UX - 本次启动)
├─ 量化标准 (度量用户看到的)
├─ 反馈 SLA (用户表达后我们接住)
├─ 周复盘 (把数字变成行动)
├─ 无障碍 (可及性底线)
├─ 应急响应 (出了事不慌)
├─ 健康度报告 (给团队/业务的反馈)
└─ 研究实验室 (让证据说话)

二者交集：
- CI 中的 monitoring-lint 接到体验维度
- PR review 含 a11y 验收清单
- 月度健康度报告联动 06 §2 5 维度
```

---

## 不允许出现的情况

- 任何用户反馈"只在群里提一句"，未落到 [feedback-pool.md](../feedback-pool.md)
- 任何 PR 不带 [a11y 验收清单](../templates/a11y-acceptance-checklist.md)
- 任何 UX-P0 事故无 [Postmortem](../templates/incident-postmortem.md)
- 任何月度健康度报告延期 ≥ 7 天
- 老年/残障专项研究连续 2 个季度空缺
- 量化标准阈值无数据佐证地修改
