# 监控指标 v1.0

> P3-14 数据库连接池监控 + P3-15 慢 SQL 监控
> 暴露方式：Spring Boot Actuator + Micrometer（Prometheus 格式）
> 安全基线：仅暴露 health/info/metrics/prometheus 四个端点，生产须在网络层限制内网可达

---

## 1. 数据库连接池（HikariCP）

通过 Spring Boot Actuator + Micrometer 暴露 Prometheus 指标。HikariCP 指标由
`DataSourcePoolMetricsAutoConfiguration` 自动绑定，无需手写 metric-registry。

### 关键指标

| 指标名 | 含义 | 告警阈值 |
|--------|------|---------|
| hikaricp_connections_active | 当前活跃连接数 | > 80% of max |
| hikaricp_connections_idle | 当前空闲连接数 | < 20% of min |
| hikaricp_connections_pending | 等待获取连接的线程数 | > 5 持续 1min |
| hikaricp_connections | 连接池总连接数 | 接近 max 时关注 |
| hikaricp_connections_max | 最大连接数（=20） | — |
| hikaricp_connections_min | 最小空闲连接数（=5） | — |
| hikaricp_connections_usage_seconds | 连接占用耗时分布 | P99 偏高需排查慢 SQL |
| hikaricp_connections_acquire_seconds | 获取连接耗时分布 | 偏高=池不足/泄漏 |

> 连接池使用率 = active / max，可在 Prometheus/Grafana 中以
> `hikaricp_connections_active{pool="micro-course-pool"} / hikaricp_connections_max{pool="micro-course-pool"}` 计算。

### 访问

```bash
curl http://localhost:8080/actuator/prometheus | grep hikaricp_
```

### 告警建议

- 持续 pending > 5：考虑扩大 maximum-pool-size
- 持续 active > 80% of max：可能存在连接泄漏，启用 HikariCP leak-detection-threshold
- 连接获取超时（acquire_seconds 飙升）：优先排查慢 SQL（见 §2）

## 2. 慢 SQL 监控

由 `MybatisSlowSqlInterceptor`（MyBatis 拦截器）实现，覆盖 query / update / batch 三类
StatementHandler 调用，监控选课、视频学习、答题等高频业务路径。

### 阈值配置

```yaml
mybatis:
  slow-sql:
    threshold-ms: 500   # 业务建议 500ms（可经 SLOW_SQL_THRESHOLD_MS 环境变量覆盖）
    alert-enabled: true # 可经 SLOW_SQL_ALERT_ENABLED 关闭
```

### 日志格式

```
[SLOW SQL] duration=750ms (threshold=500ms), sql=SELECT ... FROM enrollments WHERE ..., params=...
```

### 设计保障（UX 零退化）

- 仅在 SQL 执行前后各取一次时间戳，开销 < 1ms，对主流程零感知。
- 监控逻辑全程 try-catch 隔离，反射取 SQL 失败时降级记录，绝不向上抛出污染业务结果。
- 业务 SQL 异常按原样抛出，拦截器不吞业务异常、不改变事务行为。

### 告警升级

- 1min 内 > 10 条慢 SQL：P2 告警
- 单条 > 2000ms：P1 告警
- 单条 > 5000ms：P0 立即响应

### 优化建议

1. 加索引（参考既有 migration 中为高频查询补的索引）
2. 优化查询（避免 SELECT *、避免无界范围扫描）
3. 缓存热点数据（Redis）
4. 异步处理非关键路径

## 3. 监控接入

### 安全加固（强制）

`/actuator/prometheus` 与 `/actuator/health` 已在 SecurityConfig 放行（便于 Prometheus 无凭证抓取），
但暴露的是运行时指标，**生产环境务必在网络层限制 actuator 端点仅监控内网可达**：

- nginx 层：对 `/actuator/` 配置 `allow <监控网段>; deny all;`
- 或防火墙层：仅放行 Prometheus 抓取节点到应用端口
- 应用层已收窄 exposure.include，绝不暴露 env/beans/configprops/heapdump/threaddump

### Prometheus 抓取配置

```yaml
scrape_configs:
  - job_name: 'micro-course-api'
    metrics_path: '/actuator/prometheus'
    scrape_interval: 15s
    static_configs:
      - targets: ['micro-course-api:8080']
```

### Grafana 仪表板（建议，不强制）

建议面板：
- 连接池使用率（active / max，实时）
- 等待连接线程数（pending）
- 慢 SQL 频次（按时间分布，基于日志采集）
- 慢 SQL Top 10
- JVM 堆 / GC / HTTP 请求 P99（micrometer 自动指标）
