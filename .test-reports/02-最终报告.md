## 一、已修复 Bug（8 个）

### BUG-001 · P1-C · 数据看板 - 核心指标趋势图始终不渲染
- **根因**：`loadTrends()` 流程错——`loading=true` 时调用 `renderTrendsChart`，但 `trendsChartRef.value === null`（ref 还没挂载），函数 return 后没人再 render。
- **修复**：先 `loading=false` → `await nextTick()` → 再 render
- **影响**：admin Dashboard 第一个图（用户/课程/学员趋势折线图）
- **验证**：浏览器实测 3 个 canvas 已渲染，包含真实数据

### BUG-002 · P1-C · 数据看板 - 课程分类分布饼图始终不渲染
- **根因**：同 BUG-001（`loadCategoryStats` 同款 nextTick 错）
- **修复**：同上
- **影响**：admin Dashboard 课程分类饼图

### BUG-003 · P1-C · 数据看板 - 课程分类分布 数据结构不匹配
- **根因**：`loadCategoryStats:607` 用 `Object.entries(res.data)` 处理**数组**（后端返回 `[{status, count}, ...]`），对数组用 Object.entries 会把下标当 key
- **修复**：改为 `res.data.map(item => ({name: item.status, value: item.count}))`
- **影响**：即使 BUG-002 修好，图例也会显示 "0: [object Object]" 之类的乱码

### BUG-004 · P1-C · 数据看板 - 最近 30 天活跃趋势图始终不渲染
- **根因**：同 BUG-001
- **修复**：同上
- **影响**：admin Dashboard 最近 30 天活跃折线图

### BUG-006 · P1-C · 课程管理 - /admin/courses 路由不存在
- **根因**：`router/index.js` 没有 `/admin/courses`，只有 `/courses`，但其他 admin 路由都是 `/admin/*` 前缀
- **修复**：新增 7 个 redirect：`/admin/courses`、`/admin/courses/create`、`/admin/courses/:id`、`/admin/courses/:id/edit`、`/admin/courses/review`、`/admin/videos`、`/admin/chapters`
- **影响**：直接 URL 访问管理后台路径不再 404

### BUG-007 · P1-C · 课程管理 - 新增课程对话框在 720 高度视口下底部被裁切
- **根因**：`<el-dialog v-model="dialogVisible">` **没有 `teleport-to="body"`**，dialog 渲染在 main 容器内，main 的 `overflow: auto` 把 905px 高的 dialog 底部 293px 裁掉
- **修复**：**全局 CSS 治本**（1 处 vs 41 处）—— `.el-dialog { position: fixed; max-height: 90vh; }` + `.el-dialog__body { max-height: calc(90vh - 120px); overflow-y: auto }`
- **影响**：**41 个 el-dialog 全部受益**（确认覆盖 admin/courses/admin/teaching-classes/academic/storage-application 等全项目所有 dialog）

### BUG-009 · P1-C · 横向扫描 - nextTick 时机 Bug 至少影响 3 个 Dashboard
- **根因**：`teacher/TeacherDashboard.vue:597` 和 `academic/Dashboard.vue:462/549` 都是同款 nextTick 错（loading=true 时调 render）
- **修复**：两文件分别按相同模式改（`loading=false → nextTick → render`）
- **影响**：教师 Dashboard 2 个图 + 教务 Dashboard 2 个图

### BUG-010 · P1-C · 横向扫描 - teleport-to 缺失 Bug 影响 32 个文件
- **根因**：`grep -L "teleport" $(grep -l "<el-dialog" -r .)` 找出 41 个 vue 文件有 `<el-dialog>` 但没 `teleport` 属性
- **修复**：同 BUG-007（全局 CSS 治本）
- **影响**：所有 41 个 el-dialog（已修复）

## 四、横向扫描发现（同类问题）

### 模式 1：nextTick 时机 Bug
- **场景**：`<el-skeleton :loading="loading">` 包裹 + 内部 ECharts 容器
- **症状**：图表永远空白
- **根因**：`loadXxx()` 流程中 `loading=true` 时调用 `renderXxxChart()`，ref 没挂载
- **修复模式**：
  ```js
  async function loadXxx() {
    loading.value = true
    try {
      const data = ...
      loading.value = false       // 先关 loading
      await nextTick()             // 等 skeleton 卸载 + chart div 挂载
      renderXxxChart(data)         // 此时 ref 一定有值
    } catch (e) { ... }
  }
  ```
- **影响范围**：**4 个 Dashboard 文件**（admin/Dashboard、teacher/TeacherDashboard、academic/Dashboard、teacher/StudentGrades）

### 模式 2：el-dialog teleport 缺失 Bug
- **场景**：`<el-dialog>` 没 `teleport-to="body"`
- **症状**：长 dialog 底部被 main 容器裁切
- **根因**：main 容器 `overflow: auto`，dialog 物理上溢出视口但被裁
- **修复模式**（**全局 CSS 治本**）：
  ```css
  .el-dialog { position: fixed !important; max-height: 90vh !important; }
  .el-dialog__body { max-height: calc(90vh - 120px) !important; overflow-y: auto !important; }
  ```
- **影响范围**：**41 个 el-dialog 全部修复**

### 模式 3：后端数据结构与前端假设不匹配
- **场景**：后端返回数组，前端当对象处理
- **根因**：Vibe Coding 阶段原型用 mock object，后期接真后端没改前端代码
- **修复**：前端代码适配后端实际结构
- **案例**：BUG-003（课程分类 Object.entries）
