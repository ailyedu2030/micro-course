## P0 客户体验问题 (会直接让客户放弃/投诉/退款)

### [P0-1] 路径 U2: 购买课程后页面无明确成功提示
- **复现**:
  1. 登录 student4 (无选课用户)
  2. 课程广场 → 搜索 "Java" → 点击课程
  3. 进入课程详情页，看到"立即购买"按钮
  4. 点击"立即购买"
  5. 等待 5 秒 → 页面无明确成功/失败提示
- **截图**: `/tmp/cx-reports/5-u2-search-results.png`, `/tmp/cx-reports/6-u2-course-detail.png`
- **客户影响**: 客户不知道付款是否成功，可能重复付款或放弃购买
- **根因推测**: 后端订单创建成功但 WebSocket 推送未送达，前端无 polling fallback
- **修复建议**: 前端用 HTTP polling (每 2 秒查一次订单状态，30 秒超时)

### [P0-2] 路径 U4: 退课功能完全缺失
- **复现**:
  1. 登录 student (已有 4 门选课)
  2. 进入"我的课程"页面
  3. 找遍整个页面，没有"退课"按钮
  4. 没有"取消选课"、"退出学习"等任何相关按钮
- **截图**: `/tmp/cx-reports/14-u4-my-courses.png`
- **代码确认**: `MyCourses.vue` 全文无"退课"相关代码
- **客户影响**: 客户选错课无法退课，只能投诉或放弃
- **修复建议**: 紧急添加退课功能按钮

### [P0-3] 路径 U13: 教师无法创建新课程
- **复现**:
  1. 登录 teacher (李教授)
  2. 进入"课程管理"页面
  3. 页面显示"微专业列表"，没有"创建新课程"按钮
  4. 只有"立即申报微专业"入口
- **截图**: `/tmp/cx-reports/42-u13-course-mgmt.png`
- **代码确认**: `TeacherMicroSpecialtyList.vue` 显示教师只能管理微专业，不能创建独立课程
- **客户影响**: 教师不知道如何添加新课程，联系客服
- **修复建议**: 明确告知教师课程创建流程，或提供课程创建入口

### [P0-4] 路径 U20: 教师无法下架/重新上架课程
- **复现**:
  1. 登录 teacher
  2. 进入"课程管理"页面
  3. 找不到"下架"、"取消发布"按钮
- **截图**: `/tmp/cx-reports/67-u20-courses.png`
- **代码确认**: 教师端只有微专业管理界面，无独立课程管理
- **客户影响**: 已发布的课程无法临时下架维护
- **修复建议**: 添加课程上下架功能

### [P0-5] 路径 U23: 院系列表/专业管理/班级管理页面空白
- **复现**:
  1. 登录 admin
  2. 进入"部门管理" `/admin/departments` → 页面空白
  3. 进入"专业管理" `/admin/majors` → 页面空白
  4. 进入"班级管理" `/admin/classes` → 页面空白
- **截图**: `/tmp/cx-reports/79-u24-departments.png`, `/tmp/cx-reports/80-u24-majors.png`, `/tmp/cx-reports/81-u24-classes.png`
- **代码确认**: `src/views/admin/` 下只有 `UserList`, `OperationLogs`, `AdminSettings`, `BannerList`, `ReviewsManagement`, `Dashboard`, `TeachingClassList`，无院系/专业/班级管理页面
- **客户影响**: 管理员无法管理系统基础数据
- **修复建议**: 紧急开发院系/专业/班级管理页面

### [P0-6] 路径 U26: 微专业审核页面不存在
- **复现**:
  1. 登录 admin
  2. 访问 `/admin/microspecialties`

- **客户影响**: 微专业无法完成审核流程

## P1 客户体验问题 (操作受阻但可解决)

### [P1-1] 路径 U3: 视频学习进度保存无明确反馈
- **复现**:
  1. 登录 student → 进入课程 → 播放视频
  2. 观看 10 秒后退出
  3. 重新进入课程，视频从头开始，无"您上次看到 XX:XX"的提示
- **截图**: `/tmp/cx-reports/10-u3-learning-page.png`, `/tmp/cx-reports/11-u3-progress-check.png`
- **客户影响**: 客户以为进度没保存，但实际上已保存
- **修复建议**: 添加"上次学习到 XX:XX"提示

### [P1-2] 路径 U5: 评价功能入口不明显
- **复现**:
  1. 登录 student → 进入"我的课程"
  2. 找"写评价"按钮，找了 10 秒没找到
  3. 不知道在哪里发表课程评价
- **截图**: `/tmp/cx-reports/17-u5-my-courses.png`
- **客户影响**: 客户想评价但找不到入口，放弃评价
- **修复建议**: 在"已完成"标签页添加"待评价"标识

### [P1-3] 路径 U30: 密码错误 5 次后无锁定提示
- **复现**:
  1. 在登录页输入错误密码
  2. 连续输入 5 次
  3. 第 5 次仍然显示"密码错误"，没有"账户已锁定"提示
  4. 输入正确密码后竟然能登录成功
- **截图**: `/tmp/cx-reports/98-u30-login-page.png` 到 `/tmp/cx-reports/103-u30-attempt-5.png`
- **客户影响**: 客户担心账户安全，但不知道是否有锁定机制
- **修复建议**: 添加"密码错误 N 次，账户已锁定 X 分钟"提示

### [P1-4] 路径 U27: iPhone 视图下课程卡片布局错乱
- **复现**:
  1. iPhone 14 (375x812) 访问课程广场
  2. 课程卡片被压缩，文字重叠
  3. 视频播放按钮点击区域过小
- **截图**: `/tmp/cx-reports/90-u27-iphone-courses.png`
- **客户影响**: 移动端用户体验差
- **修复建议**: 优化响应式布局

## 根因分析

1. **功能缺失严重**: 退课、课程创建、审核等核心功能完全缺失或页面不存在
2. **移动端未优化**: iPhone/iPad 适配差，影响用户体验
3. **错误处理不完善**: 关键操作(支付、登录)的错误提示不足
4. **教师端定位模糊**: 教师端面向微专业，但课程创建等基础功能未实现
