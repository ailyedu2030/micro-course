## ADDED Requirements

### Requirement: 6 个高危联动缺陷纵向复测
系统 SHALL 对以下 6 个高危联动缺陷 (来自 2026-07-06 审计第二层跨分片随机) 做纵向串联复测,验证不出现"单按钮独立测试通过但联动失败"的情况:

1. **LD-002** 进度永久丢失 (P0)
2. **LD-005** 批量支付假成功 (P1-C)
3. **LD-006** 自审批绕过攻击链路 (P0)
4. **LD-009** 双重失效模式 (P0)
5. **LD-011** 计数器共享冲突 (P0)
6. **LD-015** 退课操作审计盲区 (P0)

#### Scenario: 串联测试执行
- **WHEN** Agent 测试任一联动缺陷
- **THEN** MUST 跑完整链路 (至少 3 个连续步骤),不允许"只测单步"

#### Scenario: 联动修复
- **WHEN** 串联测试发现任一联动缺陷
- **THEN** Agent MUST 修复根因(不是单步表现),如 LD-002 需重置 `lastReportedProgress` 状态机而非只加重试

### Requirement: 跨链路数据一致性
系统 MUST 验证跨链路数据流不出现"步骤 A 输出未被步骤 B 感知"的问题。

#### Scenario: 进度数据流
- **WHEN** 用户视频播放 30% 后断网 → 重连 → 继续播放
- **THEN** 系统 MUST 完整保留 `lastReportedProgress` 状态,不允许出现"上报失败→重置失败→进度永久丢失"

#### Scenario: 审批数据流
- **WHEN** 教师 A 提交课程审批 → 教务 B 审批通过 → 课程上架
- **THEN** 系统 MUST 确保审批数据流不出现"教师自审批"或"审批状态跨域不同步"的问题
