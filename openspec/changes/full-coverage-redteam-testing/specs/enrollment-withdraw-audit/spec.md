## ADDED Requirements

### Requirement: 退课链路全量补审
系统 SHALL 补做 2026-07-06 审计遗漏的 ENR-005/006/008/009 共 4 个退课相关操作单元,修补审计盲区 (P0-L05)。

#### Scenario: ENR-005 退课按钮
- **WHEN** 学生点击"退课"按钮
- **THEN** Agent MUST 验证: (1) 退课前确认弹窗 (2) 后端扣减优惠 (3) 触发退款 (4) 释放学习记录 (5) 通知教师

#### Scenario: ENR-006 退课确认
- **WHEN** 学生在弹窗点击"确认退课"
- **THEN** Agent MUST 验证后端事务完整性 + 退款状态机 + 学习记录软删除

#### Scenario: ENR-008 批量退课
- **WHEN** 教师或管理员触发批量退课
- **THEN** Agent MUST 验证事务边界 + 部分失败回滚 + 退款明细

#### Scenario: ENR-009 退课审计
- **WHEN** 任一退课操作完成
- **THEN** 系统 MUST 写审计日志(操作人/时间/原因/退款金额),不允许"退课无痕"

### Requirement: 退课权限校验
系统 MUST 严格按权限矩阵校验退课操作:学生仅能退自己的课、教师能退所教班级的课、管理员/教务能退任意课。

#### Scenario: 越权退课
- **WHEN** 学生 A 尝试退学生 B 的课 (伪造 studentId)
- **THEN** 系统 MUST 返回 403 错误,不允许退课成功

#### Scenario: 退课自审批
- **WHEN** 教务尝试退自己审批过的课 (关联交易)
- **THEN** 系统 MUST 拒绝或要求二次确认,不允许无痕退课
