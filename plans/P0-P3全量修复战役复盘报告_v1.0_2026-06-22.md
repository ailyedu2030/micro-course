## 四、关键发现（Cross-Agent 协调中的真相）

### 4.1 铁证级 P0 缺陷已全部修复

1. **CI 不跑 mvn test**（`.github/workflows/ci.yml:29`）→ ✅ 修复，CI 屏障立起来
2. **选课状态值非法**（`EnrollmentServiceImpl.java:114` 写入 `"ENROLLED"`）→ ✅ 修复，引入 `EnrollmentStatus` 枚举
3. **/api/files 通配 permitAll**（`SecurityConfig.java:74`）→ ✅ 修复，5 类分类授权

### 4.2 Phase D-1 发现真实数据脱节

通过 `convergence-check-status-machine.sh` 实测：
- `EnrollmentStatus` Java 7 个 vs 文档 5 个（**缺 COMPLETED, DROPPED**）
- `NotificationType` 双向脱节（Java 多 5、文档多 3）
- 这证明了文档双向校验脚本的**真实价值**

### 4.3 Phase D-2 关键偏差

任务模板假设 `UserStatus`/`TeachingClassStatus`/`OrderStatus` 等枚举"已由 Phase C-1 落地"，实测**Phase C-1 并未真正创建这些枚举**（Phase C-1 coder 报告说做了但实际未完成）。Phase D-2 自动降级，只导出真实存在的 5 个枚举。

**教训**：Phase 间依赖需要明确验证，不能仅靠前置 Agent 的报告。

### 4.4 Phase A-2 关键偏离

任务要求"不写 migration，只用 @Version 字段"，但实测 MyBatis-Plus 不会自动跳过缺失列（会导致 select 崩溃）。Phase A-2 自主决定写 V61 幂等迁移（`ADD COLUMN IF NOT EXISTS`），与 V52/V57 同款范式，零数据风险。**这是唯一既修好乐观锁又零退化的方案**。

### 4.5 Phase B-1 警告（已隔离验证）

`AuthIntegrationTest` 出现 `423 vs 401` 失败 —— 隔离复现证实为测试间状态污染（登录锁定计数器跨测试累积），非真实代码 bug。完整套件运行 94/94 PASS。

## 七、Git Commit 策略建议

### 7.1 单 commit 风险

**不建议单 commit**：
- 50+ 文件变更，单 commit 难回滚
- 测试可能因环境差异需要重跑
- code review 压力大

### 7.2 推荐分 6 个 commit

```
commit 1: fix(test): CI 跑 mvn test + 修复 6 个损坏测试种子 (Phase A-1)
commit 2: fix(state): 选课状态机枚举+乐观锁+流转白名单+审计 (Phase A-2 + V61)
commit 3: fix(security): SecurityConfig /api/files 收窄 + 端点补齐 + 权限对账 (Phase A-3 + Phase A-4)
commit 4: feat(progress): LearningProgress 乐观锁 (Phase B-1 + V62)
commit 5: feat(notification): 通知矩阵接线 5 业务事件 (Phase B-2)
commit 6: test(integration): 5 大核心链路 38 集成测试 (Phase B-3)
commit 7: fix(frontend): A11Y + 响应式 + 天气清理 (Phase B-4)
commit 8: refactor(security): PermissionEvaluator + 全局异常 + 枚举共享 (Phase C-1 + Phase D-2)
commit 9: docs(sync): 数据字典/权限矩阵/状态机文档同步 (Phase C-2)
commit 10: ci(quality): jacoco + axe-core + bundle-size 门禁 (Phase C-3)
commit 11: feat(tooling): @RequireRole + 双向校验脚本 (Phase D-1)
```

**简化方案**（推荐）：4 个 commit 合并 Round

```
commit 1: fix(p0): Phase A 止血 - CI+选课状态机+files收窄+端点补齐
commit 2: feat(p1): Phase B 业务正确性 - 进度锁+通知接线+5链路测试+前端A11Y
commit 3: refactor(p0/p1): Phase C 架构统一 + 文档同步 + CI门禁
commit 4: feat(p3): Phase D 长期工程 - 注解常量化+双向校验+枚举共享
```
